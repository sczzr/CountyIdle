# 功能卡（Feature Card）

## 功能信息

- 功能名：仓储卡槽正式贴图与短缺提示
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`WarehousePanel / MainUI / ResourceSystem / InventoryRules / assets/ui`

## 目标与用户收益

- 目标：把仓储卡槽从“文字 + emoji 占位”升级为“正式材料贴图 + 库存状态提示”，进一步强化扫视识别效率。
- 玩家可感知收益（10 分钟内）：打开仓储即可通过真实卡图识别材料类型，并快速找出当前紧缺物资。

## 实现范围

- 包含：
  - 为仓储卡槽接入正式材料图片资源
  - 为卡槽增加“充足 / 紧缺 / 缺货”状态高亮
  - 增加基础分类筛选与“紧缺”筛选入口
  - 保留现有分类分区、快捷操作与 `Esc` 关闭交互
- 不包含：
  - 不改 `GameState` 与资源公式
  - 不改仓储容量、材料链或结算顺序
  - 不增加拖拽整理、分页和自定义排序

## 实现拆解

1. 新增仓储材料贴图资源
2. 更新 `WarehousePanel` 数据定义，挂接图片路径与短缺阈值
3. 在弹窗顶部增加分类筛选与“紧缺”视图
4. 为卡槽刷新逻辑增加状态标签与高亮样式
5. 构建验证并回写结果日志

## 验收标准（可测试）

- [ ] 卡槽显示正式图片资源，不再只依赖 emoji 文本
- [ ] 玩家可切换“全部 / 各分类 / 紧缺”视图
- [ ] 紧缺与缺货卡槽能通过颜色或标签明显区分
- [ ] 现有快捷操作、提示条与 `Esc` 关闭不回归
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：正式贴图数量增加后，若路径或导入状态异常，可能出现空贴图回退。
- 回滚方式：回退到上一版卡槽配置与贴图引用逻辑，保留文字卡槽结构。

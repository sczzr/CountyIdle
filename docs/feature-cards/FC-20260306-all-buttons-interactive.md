# 功能卡（Feature Card）

## 功能信息

- 功能名：主场景按钮全量可交互修复
- 优先级：`P0`
- 目标版本：`v0.x`
- 关联系统：`UI`、`scripts/Main.cs`、`scripts/systems/CountyTownMapViewSystem.cs`

## 目标与用户收益

- 目标：消除主场景中“可见按钮但无响应/不可点”的交互断点。
- 玩家可感知收益（10 分钟内）：所有可见按钮点击后都有明确反馈（状态变化或日志提示），不再出现“按钮像装饰”的体验。

## 实现范围

- 包含：
  - 为岗位优先按钮补齐点击行为与文案反馈
  - 为矿坑卡片按钮补齐点击行为（锁定提示/解锁后执行）
  - 清理矿坑按钮默认禁用态，确保 UI 可点
- 不包含：
  - 新增科技树与矿业独立系统
  - 大规模 UI 结构重排

## 实现拆解

1. 盘点主场景所有按钮并标记未绑定入口
2. 在 `Main.cs` 补齐遗漏按钮事件与状态刷新
3. 进行构建验证并记录交互覆盖结果

## 验收标准（可测试）

- [ ] 旧布局岗位优先按钮点击后有实时反馈
- [ ] 矿坑按钮在锁定/解锁状态下均可点击并给出有效结果
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：新增按钮行为可能与原有文案预期存在轻微偏差。
- 回滚方式：回退 `scripts/Main.cs` 与 `scenes/ui/WorldPanel.tscn` 本次改动。

# 功能卡（Feature Card）

## 功能信息

- 功能名：UI / 地图表现层 GDScript 第三批拆分
- 优先级：`P1`
- 目标版本：`当前迭代`
- 关联系统：`SettingsPanel`、`SectOrganizationPanel`、`Main` 顶部地图标签栏

## 目标与用户收益

- 目标：继续将纯表现反馈从 `C#` 下放到 `GDScript`，让设置卷、宗门组织谱与中部地图顶部标签栏在不动权威逻辑的前提下更清晰。
- 玩家可感知收益（10 分钟内）：设置卷开场与录键状态更明确，宗门组织谱切峰/切职司反馈更顺手，地图顶部标签栏切页更有层次。

## 实现范围

- 包含：
  - 为 `SettingsPanel` 接入开场与快捷键录入高亮表现脚本；
  - 为 `SectOrganizationPanel` 接入开场、峰脉切换与职司卡切换表现脚本；
  - 为 `Main` 中部地图顶部标签栏接入按钮行强调反馈脚本；
  - 补齐场景节点与 `C# -> GDScript` 的单向调用边界；
  - 回写 `docs/05_feature_inventory.md` 与 `docs/08_development_list.md`。
- 不包含：
  - 不改客户端设置保存逻辑、宗门治理规则、地图选择权威逻辑；
  - 不改 `GameState`、存档结构与小时结算公式；
  - 不引入 `C++ / Rust` 运行时扩展。

## 实现拆解

1. 新增 `SettingsPanelVisualFx.gd`、`SectOrganizationPanelVisualFx.gd`、`MainTopTabVisualFx.gd`
2. 在对应 `.tscn` 中挂接辅助节点，保持权威脚本仍为 `C#`
3. 由 `C#` 在面板打开、切峰、切职司、录键状态切换、地图顶部标签切换时单向调用表现脚本

## 验收标准（可测试）

- [ ] 打开设置卷时有轻量开场反馈，录入快捷键时对应按钮有高亮反馈
- [ ] 打开宗门组织谱时有轻量开场反馈，切换峰脉与职司卡时有明确视觉反馈
- [ ] 中部地图顶部标签切换时，按钮行与当前页签有轻量强调反馈
- [ ] `GDScript` 不直接修改设置持久化、宗门规则、地图选择与核心公式
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：动态生成的宗门组织谱卡片路径不稳定时，表现脚本可能失效；设置卷录键状态如果频繁切换，可能打断 tween
- 回滚方式：移除新增 `GDScript` 节点与调用，保留现有 `C#` 逻辑链即可恢复

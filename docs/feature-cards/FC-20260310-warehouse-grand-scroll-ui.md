# 功能卡（Feature Card）

## 功能信息

- 功能名：库藏大卷全屏仓储 UI 重构
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`WarehousePanel / MainWarehousePanel / PopupPanelBase`

## 目标与用户收益

- 目标：将仓储弹窗重构为“宗门库藏大卷”全宽视图，拆分并隐藏右侧建造/生产操作，集中展示库存；以刻度进度条强化满载风险；物品卡片 token 化、数量优先级最高。
- 玩家可感知收益（10 分钟内）：一屏看清多数物资与仓储压力，减少滚动与误判。

## 实现范围

- 包含：
  - 仓储弹窗宽度扩展，库存区独占画幅
  - 4~6 列自适应网格（窄屏降列）
  - 仓储满载进度条（刻度 + 朱红填充）
  - 物品卡片 token 化、数量大号右对齐、零库存灰态
- 不包含：
  - 仓储容量/产出/结算机制变更
  - 新增仓储玩法或资源字段
  - 右侧建造/生产逻辑重写（仅表现层隐藏）

## 实现拆解

1. 重构 `WarehousePanel.tscn` 布局与进度条结构
2. 更新 `WarehousePanel.cs` 卡片生成、栅格列数与容量条样式
3. 同步 `docs/05_feature_inventory.md` 表现层补记
4. 本地验证 `dotnet build .\Finally.sln`

## 验收标准（可测试）

- [ ] 仓储弹窗宽屏呈现，库存区独占画幅
- [ ] 物品卡片 4~6 列自适应，数量右侧大号展示
- [ ] “将满/偏满”有刻度进度条可视化
- [ ] 零库存物资明显灰态处理
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：隐藏右侧操作区可能降低建造/生产入口可发现性；宽屏布局在小分辨率可能挤压。
- 回滚方式：回退 `WarehousePanel.tscn` 与 `WarehousePanel.cs` 至旧版布局，保留仓储逻辑与入口不变。

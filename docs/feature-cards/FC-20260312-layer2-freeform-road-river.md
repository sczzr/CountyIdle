# 功能卡（Feature Card）

## 功能信息

- 功能名：L2 道路与河流改为独立绘制装饰层
- 优先级：`P1`
- 目标版本：2026-03-12
- 关联系统：`docs/11_map_asset_production_spec.md`、`docs/14_map_layer_rendering_implementation_plan.md`、`docs/15_map_layer2_nanobanana_prompt_spec.md`

## 目标与用户收益

- 目标：将地图 `Layer 2` 中的道路与河流从“hex 连接片”思路调整为“独立卷轴笔触资源”，不再要求单个资源锚定六边形框架。
- 玩家可感知收益（10 分钟内）：道路与河流会更像画在卷轴上的自然笔触，而不是硬贴在 hex 中心向六边扩散的程序连接件，地图观感更松弛、更接近国风长卷。

## 实现范围

- 包含：
  - 回写 L2 道路 / 河流的资产职责、绘制边界与验收标准
  - 将 Nano Banana 提示词从 `4x2 hex connector sheet` 改为“单条道路 / 单条河流”资产
  - 更新开发看板与实现状态，避免后续继续按 hex connector 方向推进
- 不包含：
  - 本轮不改 `Layer 1` 基础地块
  - 本轮不改 `Layer 3` 立体物件
  - 本轮不直接实现新的运行时代码接入

## 实现拆解

1. 为 L2 道路 / 河流新增“独立笔触装饰层”文档口径。
2. 明确它们不与六边形格位一一绑定，只需服务地图整体视觉引导。
3. 将旧的 hex connector 方案保留为历史兼容思路，不再作为当前默认生产方向。

## 验收标准（可测试）

- [ ] `docs/11_map_asset_production_spec.md` 明确道路 / 河流不再要求按 hex 连接片生产。
- [ ] `docs/14_map_layer_rendering_implementation_plan.md` 明确道路 / 河流可自由摆放，不再以 hex 邻接为默认驱动。
- [ ] `docs/15_map_layer2_nanobanana_prompt_spec.md` 改为输出单条道路与单条河流提示词。

## 风险与回滚

- 风险：
  - 若后续运行时仍沿用旧的邻接贴图逻辑，文档与代码会暂时存在过渡差异。
  - 自由摆放方案若缺少锚点与遮挡约束，后续接入时可能出现压高亮或遮挡建筑的问题。
- 回滚方式：
  - 保留既有 hex connector 文档记录作为兼容方案；
  - 若后续验证发现自由摆放不适合当前运行时，可单独为世界图或局部图恢复连接片子方案，但不影响本轮宗门图美术生产方向。

# 功能卡（Feature Card）

## 功能信息

- 功能名：郡学突破（科研里程碑）
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`Economy / Population / GameLoop / UI`

## 目标与用户收益

- 目标：把科研点从“纯累计数字”改为分段突破增益，提升“士”职业的决策价值。
- 玩家可感知收益（10 分钟内）：约 10 分钟内可触发首个科技突破，看到粮食产出提升与突破日志。

## 实现范围

- 包含：
  - `GameState` 增加科技阶与倍率状态字段（可存档）
  - 新增 `ResearchSystem` 负责科技阶判定与倍率应用
  - `GameLoop` 接入科技突破结算与日志广播
  - `EconomySystem/PopulationSystem` 使用科技倍率计算产出与增长
  - `Main` 顶部摘要显示科技阶
- 不包含：
  - 科技树 UI 面板
  - 玩家手动点选科技分支
  - 存档格式版本迁移工具

## 实现拆解

1. 先补系统规格中的科技里程碑公式与边界
2. 按 `models → systems → core → UI` 完成最小闭环
3. 构建验证并记录平衡日志

## 验收标准（可测试）

- [x] 科研达到阈值时自动提升科技阶并产生日志
- [x] 科技加成正确作用于粮食/木石/金币/人口增长
- [ ] 读档后科技阶与倍率状态保持一致
- [x] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：早期滚雪球过快导致资源压力降低。
- 回滚方式：移除 `ResearchSystem` 调度并将倍率字段回退为固定 `1.0`。

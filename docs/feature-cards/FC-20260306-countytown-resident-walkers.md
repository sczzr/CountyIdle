# 功能卡（Feature Card）

> 历史兼容说明（2026-03-08）：本文保留旧阶段的“县城 / 郡图 / 郡县 / 州府”等表述以便追溯；在当前项目语义中，请分别按“天衍峰驻地 / 江陵府外域 / 天衍峰经营 / 世界地图”理解。
> 当前对外语义：宗门弟子可视移动 / 天衍峰山门图活化


## 功能信息

- 功能名：宗门弟子可视移动系统
- 优先级：`P2`
- 目标版本：`v0.x`
- 关联系统：`CountyTownMapViewSystem / TownMapGeneratorSystem / Main / GameState`

## 目标与用户收益

- 目标：让天衍峰山门图从静态建筑展示升级为“弟子可见、持续移动”的活地图。
- 玩家可感知收益（10 分钟内）：打开天衍峰山门图即可看到不同职司弟子在住房与工作地之间往返，宗门更有生命感。

## 实现范围

- 包含：
  - 为天衍峰山门图生成可见弟子形象（使用运行时生成的人形贴图）
  - 根据岗位结构生成不同类型弟子
  - 弟子在住房入口与工作地道路点之间循环移动
  - 地图重建与状态刷新时同步重建弟子队列
- 不包含：
  - 个体级寻路与碰撞避让
  - 角色战斗、交互、点击详情面板
  - 新增外部人物美术资源包

## 实现拆解

1. 在地图视图层定义弟子可视数据与运行时贴图
2. 按岗位结构与住房分布生成弟子移动队列
3. 在天衍峰山门图 `_Process/_Draw` 中实现动画更新与渲染
4. 从 `Main` 的状态刷新链路接入弟子更新
5. 编译验证并记录交付日志

## 验收标准（可测试）

- [ ] 天衍峰山门图上能看到多个弟子形象持续移动
- [ ] 弟子会在住房入口与工作地之间往返，而非静止不动
- [ ] 岗位变化或地图重建后，弟子数量与分布会刷新
- [ ] 弟子系统不修改经济与人口核心结算
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：弟子数量过多可能导致天衍峰山门图绘制开销上升。
- 回滚方式：保留地图与人口数据，停用弟子绘制与动画更新，仅保留静态天衍峰山门图。

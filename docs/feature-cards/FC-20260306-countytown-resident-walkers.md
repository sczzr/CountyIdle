# 功能卡（Feature Card）

## 功能信息

- 功能名：县城居民可视移动系统
- 优先级：`P2`
- 目标版本：`v0.x`
- 关联系统：`CountyTownMapViewSystem / TownMapGeneratorSystem / Main / GameState`

## 目标与用户收益

- 目标：让县城地图从静态建筑展示升级为“居民可见、持续移动”的活地图。
- 玩家可感知收益（10 分钟内）：打开县城地图即可看到不同职业居民在住房与工作地之间往返，县城更有生命感。

## 实现范围

- 包含：
  - 为县城地图生成可见居民形象（使用运行时生成的人形贴图）
  - 根据岗位结构生成不同类型居民
  - 居民在住房入口与工作地道路点之间循环移动
  - 地图重建与状态刷新时同步重建居民队列
- 不包含：
  - 个体级寻路与碰撞避让
  - 角色战斗、交互、点击详情面板
  - 新增外部人物美术资源包

## 实现拆解

1. 在地图视图层定义居民可视数据与运行时贴图
2. 按岗位结构与住房分布生成居民移动队列
3. 在县城地图 `_Process/_Draw` 中实现动画更新与渲染
4. 从 `Main` 的状态刷新链路接入居民更新
5. 编译验证并记录交付日志

## 验收标准（可测试）

- [ ] 县城地图上能看到多个居民形象持续移动
- [ ] 居民会在住房入口与工作地之间往返，而非静止不动
- [ ] 岗位变化或地图重建后，居民数量与分布会刷新
- [ ] 居民系统不修改经济与人口核心结算
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：居民数量过多可能导致县城地图绘制开销上升。
- 回滚方式：保留地图与人口数据，停用居民绘制与动画更新，仅保留静态县城地图。

# 功能卡（Feature Card）

## 功能信息

- 功能名：事件日志书卷化（札记卷）
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`EventLogPanel / MainSectChroniclePanel / Main.cs`

## 目标与用户收益

- 目标：将右侧事件日志面板统一到“浮云宗书卷侧栏”规范，用宗门札记卷替代偏现代的日志窗题头与卡片语气。
- 玩家可感知收益（10 分钟内）：右栏风闻与日志会更自然地融入整体卷轴主界面，不再出现“郡县志略”这类旧语义或现代浮窗感。

## 实现范围

- 包含：
  - 调整 `EventLogPanel.tscn` 的纸面、分隔线与风闻卡样式
  - 将标题与示例文案切到“山门近闻 / 天衍峰札记”语义
  - 保持 `AlertItem1 / AlertItem2 / LogLabel` 的现有节点路径不变
- 不包含：
  - 不改日志生成链路、风闻实时摘要规则和 `Main.cs` 绑定逻辑
  - 不改小时结算日志内容来源

## 实现拆解

1. 重绘 `EventLogPanel.tscn` 的纸面边饰与札记卷题头
2. 收敛风闻卡与正文纸框样式
3. 保持绑定路径不变并同步文案
4. 同步 `docs/02/05/08` 的右栏日志描述
5. 执行 `dotnet build .\\Finally.sln`

## 验收标准（可测试）

- [ ] 右栏日志区标题、风闻卡和正文纸框符合书卷札记风格
- [ ] `AlertItem1 / AlertItem2 / LogLabel` 路径保持兼容
- [ ] `dotnet build .\\Finally.sln` 通过

## 风险与回滚

- 风险：若右栏分隔线和风闻卡过重，可能再次抢占中央沙盘注意力。
- 回滚方式：回退 `EventLogPanel.tscn` 的题头、卡片与纸面样式改动，保留绑定路径与日志逻辑。

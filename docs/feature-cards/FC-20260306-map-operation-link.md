# 功能卡：DL-013 地图与经营状态联动（一期）

> 历史兼容说明（2026-03-08）：本文保留旧阶段的“县城 / 郡图 / 郡县 / 州府”等表述以便追溯；在当前项目语义中，请分别按“天衍峰驻地 / 江陵府外域 / 天衍峰经营 / 世界地图”理解。
> 当前对外语义：天衍峰山门图 / 江陵府外域 / 世界地图与经营联动


## 功能信息

- 功能名：地图与经营状态联动（一期）
- 优先级：`P2`
- 目标版本：`2026.03 Legacy UI Sprint`
- 关联系统：`StrategicMapViewSystem`、`CountyTownMapViewSystem`、`GameLoop`、`WorldPanel`

## 目标与用户收益

- 目标：让经营指标不再只停留在数值面板，而是同步映射到江陵府外域 / 天衍峰山门图表现；同时允许玩家从地图侧发起调度，反向影响经营状态。
- 玩家可感知收益（10 分钟内）：
  - 能从地图颜色、标题状态、提示文案直接看出“繁荣 / 平稳 / 吃紧 / 紧张”；
  - 能在地图页直接执行灵道整修、抚恤附庸、坊路修整、夜巡护山等调度；
  - 执行调度后，民心、威胁、通勤、资源会立刻变化并反映回地图。

## 实现范围

- 包含：
  - 世界/郡图/县城地图的经营状态视觉联动；
  - 地图页专用状态条与双按钮调度入口；
- 地图调度对 `Happiness`、`Threat` 与资源库存的即时影响；
  - 日志与状态刷新闭环。
- 不包含：
  - 新地图页；
  - 复杂随机地图事件链；
  - Figma 布局专用地图调度入口；
  - 月度/季度级长期地图 Buff 系统。

## 实现拆解

1. 补地图经营联动规格、功能卡与改动提案。
2. 新增地图联动系统，统一计算地图状态样式与调度效果。
3. 将状态样式接入 `StrategicMapViewSystem` 与 `CountyTownMapViewSystem`。
4. 在 `WorldPanel` 增加地图状态条与调度按钮，并由 `Main` 负责 UI 绑定。
5. 通过 `GameLoop` 暴露地图调度调用入口，保证日志与状态发布一致。

## 验收标准（可测试）

- [ ] 县城/郡图/世界图会随民心、威胁、住房/通勤等经营状态改变显示状态文案与视觉色调。
- [ ] 在郡图或县城地图页可执行至少两类地图调度，且成功/失败均有日志反馈。
- [ ] 地图调度会即时影响至少两项经营指标，且不会产生负值异常。
- [ ] `dotnet build .\Finally.sln` 通过，`60 分钟`结算节奏不变。

## 风险与回滚

- 风险：地图状态色调过强可能影响当前美术可读性；地图调度若成本过低可能被频繁刷用。
- 回滚方式：移除地图调度入口并恢复地图视图默认颜色，保留现有地图渲染与经营主循环。

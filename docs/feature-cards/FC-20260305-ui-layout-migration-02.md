# 功能卡（Feature Card）

## 功能信息

- 功能名：主界面 UI 从 `Main.cs` 拆分到 `Main.tscn`
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`UI`、`core/GameLoop`（仅绑定层）

## 目标与用户收益

- 目标：将 UI 结构与布局从代码构建改为场景资源定义，降低 `Main.cs` 复杂度并提升可视化编辑效率。
- 玩家可感知收益（10 分钟内）：界面行为保持一致，后续 UI 调整更稳定更快速。

## 实现范围

- 包含：
  - 在 `Main.tscn` 中定义现有主界面节点树与按钮
  - `Main.cs` 改为节点获取、事件绑定、状态刷新
  - 保持现有玩法逻辑与结算节奏不变
- 不包含：
  - 新增玩法机制
  - 视觉风格重做
  - 平衡参数调整

## 实现拆解

1. 先在场景中还原当前 UI 层级和关键节点命名
2. 将 `Main.cs` 的 `BuildUi`/`AddJobRow` 迁移为节点绑定与按钮事件注册
3. 构建验证并确认主循环、存读档、探险开关行为一致

## 验收标准（可测试）

- [x] `Main.tscn` 可独立承载主界面，无需运行时动态创建控件
- [x] `Main.cs` 仅处理 UI 绑定和交互，不包含复杂 UI 构建逻辑
- [x] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：场景节点名与脚本路径不一致会导致运行时 `GetNode` 报错。
- 回滚方式：回退 `Main.tscn` 与 `Main.cs` 到上一提交，恢复代码构建 UI 方式。

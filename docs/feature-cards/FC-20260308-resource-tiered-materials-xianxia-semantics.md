# 功能卡（Feature Card）

## 功能信息

- 功能名：资源系统分层扩展（V1 六期：修仙材料语义化）
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`MaterialRules / ResourceSystem / IndustrySystem / PopulationSystem / WarehousePanel / Main / PrefectureMapGeneratorSystem`

## 目标与用户收益

- 目标：在不改动底层库存字段与存档兼容的前提下，把 `T0/T1` 玩家可见材料名称、日志与地图标注统一为修仙宗门语义。
- 玩家可感知收益（10 分钟内）：玩家打开仓储、查看顶部资源条、阅读小时结算日志与外域原料标注时，会立即感知到“灵木 / 灵草 / 玄铁矿 / 养气散 / 护山构件”等统一的修仙材料体系。

## 实现范围

- 包含：
  - 新增玩家可见材料命名规则与集中映射
  - 仓储面板材料名称、描述、分类文案统一到修仙语义
  - 小时结算日志、人口缺口提示、工坊制造日志统一改用修仙材料名
  - 主界面资源条与外域原料来源标签同步切换为修仙材料名
  - 保持 `GameState` 与存档字段名不变，仅改玩家可见语义层
- 不包含：
  - 不新增新的 `T0/T1/T2/T3` 材料字段
  - 不改资源产量公式、仓储容量公式、人口消耗公式
  - 不引入丹药炼制、法宝打造等新增玩法闭环
  - 不重命名现有 `County* / Prefecture* / GameState` 技术字段

## 实现拆解

1. 建立材料语义映射，统一玩家可见名称与说明
2. 接入仓储、主界面、日志与外域原料标签
3. 回写 `DL-023` 文档、构建验证并沉淀日志

## 验收标准（可测试）

- [ ] 仓储面板中的 `T0/T1` 材料名称与描述改为修仙风格，且不影响库存显示
- [ ] `ResourceSystem / IndustrySystem / PopulationSystem` 的玩家可见日志改用统一材料名
- [ ] 外域原料来源标签与主界面材料提示切换为修仙语义
- [ ] `GameState` 兼容字段、存档结构与结算节奏保持不变
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：若命名过于抽象，可能降低玩家对材料功能的直观理解；若字符串散落改动过多，可能出现局部命名不一致。
- 回滚方式：保留集中语义映射入口；若某批名称反馈不佳，可只回滚映射表，不回退底层资源链实现。

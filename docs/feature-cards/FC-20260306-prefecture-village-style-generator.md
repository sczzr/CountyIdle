# 功能卡（Feature Card）

> 历史兼容说明（2026-03-08）：本文保留旧阶段的“县城 / 郡图 / 郡县 / 州府”等表述以便追溯；在当前项目语义中，请分别按“天衍峰驻地 / 江陵府外域 / 天衍峰经营 / 世界地图”理解。
> 当前对外语义：江陵府外域 / 外域附庸据点程序化生成


## 功能信息

- 功能名：江陵府外域 Village 风格程序化生成
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`StrategicMap`、`UI`

## 目标与用户收益

- 目标：将“江陵府外域”从静态配置轮廓升级为可随宗门经营状态变化的程序化附庸据点地图。
- 玩家可感知收益（10 分钟内）：每次经营阶段推进后，外域道路、附庸据点节点与边界形态会出现可感知变化，地图更像“活着的外域”。

## 实现范围

- 包含：
  - 新增江陵府外域程序化生成系统（有机边界、道路骨架、河流与附庸据点节点）
  - `Prefecture` 模式渲染接入程序化结果，`World` 模式保持现状
  - 按人口/住房/威胁/结算进度分桶重建，避免每次状态广播都重绘
- 不包含：
  - 不接入外部网站源码或运行时代码注入
  - 不改世界地图 JSON 配置结构
  - 不改核心结算顺序与存档结构

## 实现拆解

1. 设计并实现 `PrefectureMapGeneratorSystem`（输入状态提示，输出 `StrategicMapDefinition`）。
2. 改造 `StrategicMapViewSystem` 仅在 `Prefecture` 模式启用程序化生成。
3. 在 `Main` 中增加江陵府外域刷新调用并执行构建验证。

## 验收标准（可测试）

- [ ] 江陵府外域显示程序化生成的边界/道路/河流/节点，不再依赖固定一组静态点位
- [ ] 人口或威胁分桶变化后，江陵府外域会重建；未变化时不重复重建
- [ ] 天下外域（World）地图渲染保持不回退
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：程序化参数波动过大可能导致江陵府外域抖动感强、可读性下降。
- 回滚方式：回退 `StrategicMapViewSystem` 与新增生成系统，恢复静态配置渲染。

# 功能卡（Feature Card）

## 功能信息

- 功能名：周边郡图 Village 风格程序化生成
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`StrategicMap`、`UI`

## 目标与用户收益

- 目标：将“周边郡图”从静态配置轮廓升级为可随县域状态变化的程序化聚落地图。
- 玩家可感知收益（10 分钟内）：每次经营阶段推进后，郡图道路、村镇节点与边界形态会出现可感知变化，地图更像“活着的郡县”。

## 实现范围

- 包含：
  - 新增周边郡图程序化生成系统（有机边界、道路骨架、河流与聚落节点）
  - `Prefecture` 模式渲染接入程序化结果，`World` 模式保持现状
  - 按人口/住房/威胁/结算进度分桶重建，避免每次状态广播都重绘
- 不包含：
  - 不接入外部网站源码或运行时代码注入
  - 不改世界地图 JSON 配置结构
  - 不改核心结算顺序与存档结构

## 实现拆解

1. 设计并实现 `PrefectureMapGeneratorSystem`（输入状态提示，输出 `StrategicMapDefinition`）。
2. 改造 `StrategicMapViewSystem` 仅在 `Prefecture` 模式启用程序化生成。
3. 在 `Main` 中增加郡图刷新调用并执行构建验证。

## 验收标准（可测试）

- [ ] 周边郡图显示程序化生成的边界/道路/河流/节点，不再依赖固定一组静态点位
- [ ] 人口或威胁分桶变化后，周边郡图会重建；未变化时不重复重建
- [ ] 天下州域（World）地图渲染保持不回退
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：程序化参数波动过大可能导致郡图抖动感强、可读性下降。
- 回滚方式：回退 `StrategicMapViewSystem` 与新增生成系统，恢复静态配置渲染。

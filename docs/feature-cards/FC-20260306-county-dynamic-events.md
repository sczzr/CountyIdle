# 功能卡（Feature Card）

> 历史兼容说明（2026-03-08）：本文保留旧阶段的“县城 / 郡图 / 郡县 / 州府”等表述以便追溯；在当前项目语义中，请分别按“天衍峰驻地 / 江陵府外域 / 天衍峰经营 / 世界地图”理解。
> 当前对外语义：宗门见闻 / 山门动态事件


## 功能信息

- 功能名：宗门见闻（动态事件）
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`GameState / CountyEventSystem / GameLoop / Main日志`

## 目标与用户收益

- 目标：让职司分配与威胁状态在 `1~5` 小时结算窗口内产生可见的宗门见闻反馈。
- 玩家可感知收益（10 分钟内）：可在日志中看到事件播报，并观察资源/幸福即时波动。

## 实现范围

- 包含：
  - `GameState` 新增事件冷却字段（可存档）
  - 新增 `CountyEventSystem`，实现候选事件判定、权重抽取、效果结算
  - `GameLoop` 接入事件系统并发布日志
- 不包含：
  - 事件手动选择分支 UI
  - 事件图鉴与历史面板
  - 多阶段链式事件

## 实现拆解

1. 先补 `docs/02_system_specs.md` 中事件系统规格与边界
2. 按 `models → systems → core` 接入动态事件最小闭环
3. 本地构建验证并补平衡日志（前后差异与结果）

## 验收标准（可测试）

- [ ] 满足事件条件时可触发至少 1 类正向事件并产生日志
- [ ] 高威胁状态可触发袭扰事件并正确扣减资源（不出现负值）
- [ ] 事件冷却按小时递减，旧存档读档不崩溃
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：事件触发过频可能掩盖基础结算反馈，或导致资源波动过大。
- 回滚方式：移除 `CountyEventSystem` 调度，删除 `EventCooldownHours` 读写依赖并恢复旧日志流。

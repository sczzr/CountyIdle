# 功能卡（Feature Card）

## 功能信息

- 功能名：UI / 地图表现层 GDScript 第二批拆分
- 优先级：`P1`
- 目标版本：`当前迭代`
- 关联系统：`TaskPanel`、`DisciplePanel`、`WorldPanel`

## 目标与用户收益

- 目标：延续第一批 `C# 管规则 / GDScript 管表现` 边界，把宗主管理册、弟子谱与中部地图页签的纯表现反馈继续下放到 `GDScript`。
- 玩家可感知收益（10 分钟内）：治宗册切页、弟子谱筛选与详情切换、世界 / 山门 / 二级地图切页反馈更顺滑，界面响应更明确。

## 实现范围

- 包含：
  - 为 `TaskPanel` 接入开场、页签切换与庶务详情脉冲表现脚本；
  - 为 `DisciplePanel` 接入开场、筛选刷新与详情卡片切换表现脚本；
  - 为 `WorldPanel` 接入中部地图页签切换与二级地图检视面板脉冲表现脚本；
  - 补齐场景节点与 `C# -> GDScript` 的单向调用边界；
  - 回写 `docs/05_feature_inventory.md` 与 `docs/08_development_list.md`。
- 不包含：
  - 不改门人、治理、地图生成、存档与小时结算公式；
  - 不改 `GameLoop`、`GameState`、`SaveSystem`、地图选择权威逻辑；
  - 不引入 `C++ / Rust` 运行时扩展。

## 实现拆解

1. 新增 `TaskPanelVisualFx.gd`、`DisciplePanelVisualFx.gd`、`WorldPanelVisualFx.gd`
2. 在对应 `.tscn` 中挂接辅助节点，保持权威脚本仍为 `C#`
3. 由 `C#` 在面板打开、页签切换、详情刷新、世界点位面板刷新时单向调用表现脚本

## 验收标准（可测试）

- [ ] 打开治宗册时有轻量开场反馈，切换治理页签时有轻量切页反馈，庶务详情切换时有脉冲反馈
- [ ] 打开弟子谱时有轻量开场反馈，筛选 / 排序切换后名册区有刷新反馈，选中弟子后详情卡片有切换反馈
- [ ] 中部地图从世界 / 山门 / 二级地图间切页时有轻量反馈，世界点位详情刷新时二级地图检视区有强调反馈
- [ ] `GDScript` 不直接修改 `GameState`、存档、地图选择与核心公式
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：场景节点路径变更后，表现脚本可能失效；详情切换若在隐藏态触发，可能出现无效 tween
- 回滚方式：移除新增 `GDScript` 节点与调用，保留现有 `C#` 逻辑链即可恢复

# 功能卡（Feature Card）

> 历史兼容说明（2026-03-08）：本文保留旧阶段的“县城 / 郡图 / 郡县 / 州府”等表述以便追溯；在当前项目语义中，请分别按“天衍峰驻地 / 江陵府外域 / 天衍峰经营 / 世界地图”理解。


## 功能信息

- 功能名：三层地图可视化与统一缩放
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`UI`、`地图显示`

## 目标与用户收益

- 目标：让县城图、郡图、天下州域图都在中央 `MapViewport` 实际渲染，并支持统一缩放交互。
- 玩家可感知收益（10 分钟内）：地图切页不再是占位文案，且可按需放大缩小观察细节。

## 实现范围

- 包含：
  - 世界图与郡图绘制脚本（占位图形化渲染，不是纯文本）
  - 县城图接入缩放能力
  - 顶部增加缩放控件（放大/缩小/重置）并支持鼠标滚轮
  - `Main.cs` 统一管理当前地图页缩放状态显示
- 不包含：
  - 地图玩法数据源与寻路系统
  - 探险战斗规则与事件数值设计

## 实现拆解

1. 新增战略地图渲染系统并挂载世界/郡图页
2. 为县城地图系统增加缩放参数与绘制缩放
3. 在 `TopTabRow` 增加缩放控件并在 `Main.cs` 绑定

## 验收标准（可测试）

- [ ] 三个地图页均可显示图形化地图内容
- [ ] 鼠标滚轮可缩放三种地图
- [ ] 顶部缩放按钮可对当前地图页生效，且缩放百分比正确更新
- [ ] 切换到非地图页时缩放控件自动禁用
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：缩放后绘制坐标溢出导致边缘裁切异常
- 回滚方式：回退 `WorldPanel.tscn`、`Main.cs`、新增地图脚本与本功能文档

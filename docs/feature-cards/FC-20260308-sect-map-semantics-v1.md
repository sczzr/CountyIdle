# 功能卡：宗门地图语义改造（V1）

## 功能信息

- 功能名：宗门地图语义改造（V1）
- 优先级：`P1`
- 目标版本：当前迭代（2026-03-08）
- 关联系统：`SectMapSemanticRules`、`SectMapViewSystem`、`TownMapGeneratorSystem`

## 目标与用户收益

- 目标：让宗门地图的场所命名、状态提示和脚本入口统一切到宗门语义。
- 玩家可感知收益（10 分钟内）：进入宗门地图后，能直接看到灵田、炼器坊、藏经阁、宗务殿等宗门场所，而不是县城/市集/官署语义。

## 实现范围

- 包含：
  - 新增宗门地图语义规则；
  - 统一宗门场所标签与状态提示；
  - 新增宗门地图脚本入口 `SectMapViewSystem`。
- 不包含：
  - 不重写宗门地图底层生成器坐标规则；
  - 不改经济、岗位或建筑系统的底层枚举。

## 实现拆解

1. 新增宗门语义规则映射。
2. 将驻地锚点标签切到宗门命名。
3. 将宗门地图选中提示和状态文本切到宗门语义。
4. 把场景脚本入口切到 `SectMapViewSystem`。

## 验收标准（可测试）

- [x] 宗门地图标签切换为宗门场所命名。
- [x] 宗门地图提示文本不再出现“县城 / 官署 / 市集 / 茶肆”等主地图语义。
- [x] `dotnet build .\Finally.sln` 通过。

## 风险与回滚

- 风险：底层仍沿用旧 `TownActivityAnchorType` 枚举，若后续玩法也要宗门化，需要继续补充更深层抽象。
- 回滚方式：保留 `SectMapViewSystem`，将 `SectMapSemanticRules` 调用回退到旧县城命名即可。

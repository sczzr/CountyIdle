# 功能卡（Feature Card）

## 功能信息

- 功能名：探险装备品质与词条掉落
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`systems/CombatSystem`、`systems/EquipmentSystem`、`models/GameState`、`UI`

## 目标与用户收益

- 目标：让“武装探险”产出可见的装备成长反馈，形成探险 → 装备提升 → 探险更稳的闭环。
- 玩家可感知收益（10 分钟内）：探险日志可看到装备掉落品质与词条，战备评分持续增长，后续胜率提升更明显。

## 实现范围

- 包含：
  - 新增装备掉落系统（品质权重 + 词条加成）
  - 探险胜利时触发装备掉落并提升 `AvgGearScore`
  - 按品质累计装备库存统计（普通/精良/史诗/传说）
  - Figma 面板补展示：战备评分与装备品质库存
- 不包含：
  - 装备栏位穿戴管理（头/身/武器等）
  - 词条重铸、分解、锻造台主动操作

## 实现拆解

1. 补系统规格与提案，明确掉落概率、品质权重、词条规则与边界
2. 新增 `EquipmentSystem` 并接入 `CombatSystem` 胜利流程
3. 更新主界面展示与日志反馈，完成最小可观测闭环

## 验收标准（可测试）

- [ ] 探险胜利时有概率产出装备，日志显示品质与词条结果
- [ ] `AvgGearScore` 可被掉落驱动提升，且不出现负值/异常跳变
- [ ] `GameState` 新增装备计数字段后存档读档兼容
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：掉落增益过高会压缩中期挑战窗口，导致探索风险感下降。
- 回滚方式：仅撤回 `EquipmentSystem` 接入与相关参数，保留文档记录用于二次调参。

# 功能卡（Feature Card）

## 功能信息

- 功能名：多存档槽管理界面（V1：手动槽）
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`SaveSystem / SqliteSaveRepository / MainUI / MainShortcutBindings / PopupPanelBase`

## 目标与用户收益

- 目标：在 SQLite 存档底层之上，提供可视化的多存档槽管理面板，支持查看、覆盖存档、读取、新建、重命名、删除手动槽。
- 玩家可感知收益（10 分钟内）：点击主界面“存档 / 读档”即可打开多存档面板，不再只能操作默认槽；可以把当前进度另存为多个手动存档，并按槽位读取或整理。

## 实现范围

- 包含：
  - 新增多存档槽弹窗面板
  - 主界面“存档 / 读档”按钮改为打开该面板
  - 面板支持：
    - 查看槽位列表
    - 覆盖所选槽
    - 读取所选槽
    - 新建槽并存档
    - 重命名所选槽
    - 删除所选槽（主存档除外）
    - 刷新列表
  - 快速存档 / 快速读档继续保留为默认槽直存直读
- 不包含：
  - 自动存档槽策略
  - 存档槽截图预览
  - 存档排序、搜索、标签
  - 跨设备同步与云存档

## 实现拆解

1. 扩展 `SaveSystem` 与仓库层支持槽位级操作
2. 新增 `SaveSlotsPanel` 场景与脚本
3. 将主界面 `存档 / 读档` 按钮切换为打开面板
4. 保留 `QuickSave / QuickLoad` 默认槽直读直写
5. 构建验证并回写看板状态

## 验收标准（可测试）

- [ ] 主界面点击“存档 / 读档”可打开多存档槽面板
- [ ] 面板可列出 SQLite 中现有槽位及其摘要信息
- [ ] 可将当前状态覆盖到所选槽
- [ ] 可输入名称新建一个手动槽并写入当前状态
- [ ] 可从所选槽读档并恢复 `GameState`
- [ ] 可重命名和删除自定义槽位
- [ ] 快速存档 / 快速读档仍可工作，并继续使用主存档
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：存档入口从“直接按钮”改为“弹窗面板”后，玩家操作路径变长；若槽位列表与主存档语义不清，可能产生误操作。
- 回滚方式：保留 SQLite 底层与默认槽逻辑，仅将主界面按钮切回直接存档/读档。

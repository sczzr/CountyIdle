# 功能卡（Feature Card）

## 功能信息

- 功能名：RimWorld 风格主界面多场景拆分
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`UI`

## 目标与用户收益

- 目标：将主界面从单一 `Main.tscn` 重构为多 `tscn` 组合布局（顶部信息条、左侧岗位面板、中部主视图、右侧日志、底部操作条）。
- 玩家可感知收益（10 分钟内）：信息区域更清晰，操作入口更稳定，后续 UI 迭代更快。

## 实现范围

- 包含：
  - 新增可复用 UI 子场景：`TopBar`、`JobsPanel`、`JobRow`、`WorldPanel`、`EventLogPanel`、`BottomBar`
  - `Main.tscn` 改为组合式布局并实例化以上子场景
  - `Main.cs` 改为绑定新节点路径与交互事件
- 不包含：
  - 玩法规则与数值改动
  - 存档结构调整
  - 视觉美术资源替换

## 实现拆解

1. 定义 UI 合同：显示数据、交互入口、刷新触发
2. 拆分多场景并由 `Main.tscn` 组合
3. 更新 `Main.cs` 绑定路径并保留事件驱动刷新

## 验收标准（可测试）

- [x] `Main.tscn` 不再承载完整 UI 细节，只做布局与子场景装配
- [x] 至少 4 个独立 UI 子场景（`tscn`）可单独维护
- [x] 岗位调整、探险开关、存档/读档、日志显示行为保持可用
- [x] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：节点路径更新遗漏会导致 `GetNode` 失败
- 回滚方式：回退 `Main.tscn` + `Main.cs` + `scenes/ui/*` 到上一个稳定版本

# 功能卡（Feature Card）

## 功能信息

- 功能名：仓储与矿产资源链（开采→冶炼→新材料→制造→建造）
- 优先级：`P0`
- 目标版本：`v0.x`
- 关联系统：`GameState / IndustrySystem / ResourceSystem / GameLoop / MainUI`

## 目标与用户收益

- 目标：在现有产业基础上补齐“仓储容量”和“矿产资源链”闭环。
- 玩家可感知收益（10 分钟内）：可看到仓储占用、矿石产出、新材料研发与产品/建材制造反馈，并影响建造与制工具结果。

## 实现范围

- 包含：
  - 新增仓储容量、仓储等级、容量溢出处理
  - 新增矿石资源（铁矿、铜矿、煤矿）与矿坑等级
  - 新增资源链系统（开采、冶炼、研发新材料、制造产品、制造建材）
  - 建筑建造与工具制造接入矿产资源链消耗
  - 报表按钮接入“矿仓联建”操作
  - UI 展示仓储占用与关键新资源摘要
- 不包含：
  - 独立仓库面板与详细资源图鉴
  - 分支科技树与复杂配方编辑器
  - 多地图矿点占领玩法

## 实现拆解

1. 更新系统规格（新增资源链章节与结算顺序）
2. 扩展 `GameState` 与产业规则（仓储与新资源字段）
3. 新增 `ResourceSystem` 并接入小时结算
4. 扩展 `IndustrySystem`（矿仓联建、建造与制工具新消耗）
5. 更新 `Main.cs` 展示与操作绑定
6. 构建验证并补平衡日志

## 验收标准（可测试）

- [ ] 每小时可稳定产出矿石，且不出现负资源
- [ ] 科技达到条件后可将矿石转化为新材料并用于制造
- [ ] 建筑建造与制工具可消费新建材/新产品
- [ ] 仓储超容时触发溢出处理并有日志提示
- [ ] 报表页矿坑按钮可触发矿仓联建
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：新资源链引入后，早期资源节奏可能偏紧，影响扩建体验。
- 回滚方式：临时下线 `ResourceSystem` 调度，并移除建造/制工具新增消耗项。

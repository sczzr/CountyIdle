# 功能卡（Feature Card）

## 功能信息

- 功能名：人口分配与生活循环系统（住房/通勤/生老病死）
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`GameState / PopulationSystem / IndustrySystem / EconomySystem / TownMap / Main HUD`

## 目标与用户收益

- 目标：让人口从“纯数值”升级为“有生活约束的劳动力系统”，形成可解释的增长与减员逻辑。
- 玩家可感知收益（10 分钟内）：住房不足、长通勤、衣物短缺会直接反映在到岗率、产出和日志中。

## 实现范围

- 包含：
  - 新增人口分层与健康字段（幼年/成年/老年/病患）
  - 住房-睡眠承载与衣物覆盖规则
  - 出生/衰老/患病/死亡的小时聚合结算
  - 步行通勤时长与到岗率系数
  - 情绪修正接入睡眠/衣物/通勤/疾病
- 不包含：
  - 个体寻路级（每个居民）实时路径模拟
  - 纺织产业完整生产链（先以库存字段承载）
  - 新增复杂人口面板美术（先以日志+基础字段显示）

## 实现拆解

1. 更新 `docs/02_system_specs.md`，冻结公式与边界
2. 扩展 `GameState` 字段与默认值，确保旧存档兼容
3. 在 `PopulationSystem` 接入生老病死与睡眠/衣物/情绪结算
4. 在 `EconomySystem` 以 `onDutyFactor` 替换纯岗位人数
5. 在 UI 日志增加通勤迟到、疾病波动、住房拥挤反馈

## 验收标准（可测试）

- [ ] 住房不足会使 `sleepFactor` 下降，并在 `1~2` 小时内反映为有效岗位减少
- [ ] 通勤距离变长会降低 `onDutyFactor`，且日志出现“远距通勤”提示
- [ ] 出生/衰老/死亡/患病恢复均有独立增减日志且无负值异常
- [ ] 衣物覆盖不足会提升患病率并带来幸福惩罚
- [ ] `60` 分钟小时结算频率不变
- [ ] `dotnet build .\\Finally.sln` 通过

## 风险与回滚

- 风险：参数过严可能导致人口与产业同步下滑，飞轮短期停滞。
- 回滚方式：保留新增字段，停用通勤/睡眠/衣物对岗位的乘算影响，回退到当前人口增长逻辑。

# 功能卡（Feature Card）

## 功能信息

- 功能名：仓储卡槽化库存界面
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`WarehousePanel / MainUI / ResourceSystem / InventoryRules`

## 目标与用户收益

- 目标：将现有文本型仓库清单重构为卡槽网格，让玩家在一个弹窗里直观看到具体材料图标与数量。
- 玩家可感知收益（10 分钟内）：打开仓储后可一眼识别“是什么材料、还剩多少、属于哪一类”，不再需要逐行读长文本。

## 实现范围

- 包含：
  - 将仓储库存展示改为按分类分区的滚动卡槽网格
  - 每个材料卡槽显示图标、名称与整数库存
  - 保留现有仓储负载、提示条、矿仓联建与链路扩建快捷操作
  - 零库存卡槽保持可见但弱化，便于玩家预判后续产线
- 不包含：
  - 不改 `GameState` 字段与小时结算逻辑
  - 不新增仓储筛选、排序、分页系统
  - 不改 Figma 主布局，仅调整 Legacy 弹窗

## 实现拆解

1. 更新 `docs/02_system_specs.md`、`05_feature_inventory.md`、`08_development_list.md`
2. 重构 `WarehousePanel.tscn` 为滚动库存区 + 分区卡槽布局
3. 在 `WarehousePanel.cs` 中用配置驱动生成材料卡槽并刷新数量
4. 保留原快捷操作事件与 `Esc` 关闭交互
5. 执行构建验证并补结果记录

## 验收标准（可测试）

- [ ] 仓储弹窗以卡槽网格展示基础储备、自然原料、矿石锭材、民生工坊库存
- [ ] 每个卡槽都显示具体材料图标、名称与整数数量
- [ ] 打开/刷新仓储时不报空引用，原快捷操作按钮继续可用
- [ ] `Esc` 关闭、提示条反馈、仓储负载条保持正常
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：材料卡槽数量增加后，低分辨率下需要依赖滚动区域，若布局间距不当会影响可读性。
- 回滚方式：回退 `WarehousePanel.tscn` 与 `WarehousePanel.cs` 到文本分类展示版本，保留现有弹窗入口与快捷操作逻辑。

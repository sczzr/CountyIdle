# 功能卡（Feature Card）

## 功能信息

- 功能名：地图素材生产规格与分层资产流水线（文档一期）
- 优先级：`P1`
- 目标版本：2026-03-12
- 关联系统：`SectMap`、`WorldMap`、`CountyTownMapViewSystem`、`TownMapGeneratorSystem`、`docs/02_system_specs.md`

## 目标与用户收益

- 目标：为天衍峰山门图与后续世界地图建立统一的地图素材生产规格，明确视觉基调、分层职责、命名规则、尺寸锚点、拼接逻辑与首批交付范围，避免地图表现层继续依赖零散贴图与临时口头约定。
- 玩家可感知收益（10 分钟内）：后续地图在保持卷轴修仙气质的同时，更容易读出地形差异、道路河流走向、核心据点和可交互重点，不再停留在“能用但不成画”的纯底格阶段。

## 实现范围

- 包含：
  - 新增正式文档 `docs/11_map_asset_production_spec.md`
  - 定义地图 5 层分层结构、贴图分类、变种数量、尺寸与锚点规范
  - 规定河流 / 道路 / 法阵拼接逻辑与破框遮挡原则
  - 规定目录、命名、交付包与验收口径
  - 回写 `docs/README.md`、`docs/05_feature_inventory.md`、`docs/08_development_list.md`
- 不包含：
  - 本轮不直接绘制正式素材
  - 本轮不重写 `CountyTownMapViewSystem` 渲染实现
  - 本轮不接入存档、小时结算或完整二级地图运行时

## 实现拆解

1. 对齐当前宗门图 / 世界图主线与现有文档状态，确认素材规格挂载点。
2. 编写正式地图素材生产规格文档，收口视觉基调、技术约束与首批资产包。
3. 将该规格登记到开发列表与实现看板，作为地图主线的正式配套文档。

## 验收标准（可测试）

- [ ] `docs/11_map_asset_production_spec.md` 已明确写出地图分层、资产清单、尺寸锚点、命名规则与首批交付包。
- [ ] 文档能明确回答“哪些层必须 tile 化、哪些层允许自由摆放、哪些物件允许破框”。
- [ ] 河流 / 道路 / 法阵的拼接逻辑已被收口为可程序消费的规则，而不是仅停留在视觉描述。
- [ ] `docs/README.md`、`docs/05_feature_inventory.md`、`docs/08_development_list.md` 已同步登记，后续不会出现文档孤岛。

## 风险与回滚

- 风险：
  - 若文档把表现规则写得过死，后续不同地图类型的表现差异会被过度束缚。
  - 若只谈画风不谈技术接口，后续 Godot 接入仍会回到临时对接状态。
- 回滚方式：
  - 保留“5 层分层 + 命名 / 尺寸 / 锚点 + 首批素材包”硬约束。
  - 对风格示例、首批品类数量与部分目录建议保留可调整空间，不在一期写死全部最终资产总量。

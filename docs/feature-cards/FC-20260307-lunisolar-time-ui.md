# 功能卡（Feature Card）

## 功能信息

- 功能名：架空农历与二十四节气时间展示
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`UI / GameLoop / GameCalendarSystem / CountyTownMapViewSystem`

## 目标与用户收益

- 目标：把顶部时间信息从“纪元/周期/结算倒计时”改为更贴合题材的架空农历日期，并补齐三档倍率控制。
- 玩家可感知收益（10 分钟内）：能直观看到“某年某月某日 + 当前节气”，并在 `x1/x2/x4` 之间快速切换。

## 实现范围

- 包含：
  - 新增架空农历换算与二十四节气展示
  - 移除顶部“结算倒计时”文案
  - 将时间倍率扩展为 `x1/x2/x4`
  - 让县城居民可视移动兼容 `x4`
- 不包含：
  - 不改变 `1 秒 = 1 游戏分钟`
  - 不改变 `60 游戏分钟 = 1 次小时结算`
  - 不改存档结构

## 实现拆解

1. 新增日历系统，把 `GameMinutes` 转为架空农历日期与节气信息。
2. 改造 Legacy/Figma 顶栏与底栏，移除倒计时并接入三档倍率。
3. 校正读档后的小时内进度与居民动画倍率兼容。

## 验收标准（可测试）

- [ ] 顶栏显示为“某某年 某月 某日”，并附带当前节气
- [ ] 主界面不再出现“结算倒计时”文案
- [ ] 倍率按钮与快捷键支持 `x1/x2/x4` 循环切换
- [ ] 读档后小时结算仍按原时间相位继续，不出现错位
- [ ] `dotnet build .\\Finally.sln` 通过

## 风险与回滚

- 风险：日期显示与真实结算节奏脱节，或 `x4` 下居民插值动画过快/异常。
- 回滚方式：回退 `GameCalendarSystem`、`Main.cs`、`GameLoop.cs` 与对应 UI 场景改动，恢复旧版纪元/周期展示与 `x1/x2` 倍率。

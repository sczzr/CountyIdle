# 功能卡（Feature Card）

## 功能信息

- 功能名：主界面交互重连与按钮功能闭环
- 优先级：`P0`
- 目标版本：`v0.x`
- 关联系统：`UI`、`core/GameLoop`、`systems/IndustrySystem`

## 目标与用户收益

- 目标：修复主界面“按钮可见但无响应”问题，并将关键按钮接入实际玩法逻辑。
- 玩家可感知收益（10 分钟内）：点击底栏与中部动作按钮可直接触发探险、倍速、存读档、产业扩建与制工具，日志与资源即时反馈。

## 实现范围

- 包含：
  - 为 Figma 主界面补齐按钮事件绑定（探险、倍速、存档、读档、重置）
  - 为 Figma 中央动作按钮接入产业扩建/制工具能力
  - 将 x2 速度从“视觉占位”改为真实结算倍率
  - 统一主界面状态刷新（资源、周期、日志）在两套布局下均可用
- 不包含：
  - 新增复杂建筑链或科技树机制
  - 大规模 UI 重构与资产替换

## 实现拆解

1. 梳理两套布局节点并建立统一绑定入口
2. 为缺失按钮补事件并连接既有系统接口
3. 校准状态刷新与日志输出，确保交互可见可验证

## 验收标准（可测试）

- [ ] Figma 布局底栏按钮点击后有明确功能反馈与日志输出
- [ ] Figma 布局中部三个动作按钮可触发产业相关操作
- [ ] `x2` 按钮可提升时间推进速度，不再是纯文本占位
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：双布局并行时可能出现个别文本状态不同步
- 回滚方式：将 `Main.cs` 恢复到仅旧布局绑定版本，并保留本次文档记录以便重做

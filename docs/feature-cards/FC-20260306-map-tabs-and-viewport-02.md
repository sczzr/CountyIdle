# 功能卡（Feature Card）

> 历史兼容说明（2026-03-08）：本文保留旧阶段的“县城 / 郡图 / 郡县 / 州府”等表述以便追溯；在当前项目语义中，请分别按“天衍峰驻地 / 江陵府外域 / 天衍峰经营 / 世界地图”理解。


## 功能信息

- 功能名：中心 MapViewport 填充布局与 TopTabRow 内容切换
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`UI`

## 目标与用户收益

- 目标：让 `MapViewport` 作为中部核心承载区域以 `fill` 方式铺满，并通过 `TopTabRow` 切换地图、事件、统计、探险等内容页。
- 玩家可感知收益（10 分钟内）：同一中部区域可快速切换不同信息视图，减少跳转与视觉割裂。

## 实现范围

- 包含：
  - `WorldPanel` 的 `TopTabRow` 扩展为“内容页”切换入口
  - `MapViewport` 仅保留一个可填充的内容容器并承载多页面
  - `Main.cs` 增加内容页切换绑定、按钮选中态同步
  - 保留产业升级按钮能力（迁移到统计页）
- 不包含：
  - 新增地图玩法规则或寻路逻辑
  - 事件系统/报表系统后端数据深化

## 实现拆解

1. 调整 `WorldPanel.tscn` 中部结构：TopTabRow + fill MapViewport + 内容页容器
2. 在 `MapViewport` 内加入地图/事件/统计/探险占位页并迁移原产业卡片
3. 在 `Main.cs` 绑定页签按钮，统一切页与按钮状态

## 验收标准（可测试）

- [ ] `MapViewport` 在中部区域随窗口拉伸填充显示
- [ ] 点击 `TopTabRow` 可切换“县城地图/郡县地图/天下州域/事件面板/统计报表/野外探险”
- [ ] 切换时按钮选中态与页面显示一致
- [ ] 统计页的产业升级按钮仍可触发原有逻辑
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：节点路径迁移后按钮绑定失效，导致运行时报错
- 回滚方式：回退 `CountyIdle/scenes/ui/WorldPanel.tscn` 与 `CountyIdle/scripts/Main.cs`

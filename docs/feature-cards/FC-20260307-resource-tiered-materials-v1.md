# 功能卡（Feature Card）

## 功能信息

- 功能名：资源系统分层扩展（V1：T0-T1 原材料与加工链）
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`GameState / ResourceSystem / IndustrySystem / EconomySystem / ResearchSystem / PopulationSystem / WarehousePanel / StrategicMap`

## 目标与用户收益

- 目标：将地图资源从“抽象矿产/材料”升级为“自然原材料 -> 一次加工 -> 二次加工 -> 成品”的分层体系，并先完成 `T0 + T1` 的最小闭环。
- 玩家可感知收益（10 分钟内）：玩家能清楚看懂“地图采到什么、工坊做出什么、最终拿来干什么”，并直接感知住房、药物、衣物、工具与基础冶铸的来源差异。

## 实现范围

- 包含：
  - 定义 `原材料层 / 一次加工层 / 二次加工层 / 成品层` 四层资源边界
  - 接入 `T0` 原材料：`林木 / 原石 / 黏土 / 卤水(岩盐) / 药材 / 麻料 / 芦苇 / 皮毛兽骨`
  - 接入 `T1` 原材料：`铜矿 / 铁矿`
  - 接入运行时一期兼容层：
    - `ResourceSystem` 采集、初加工、民生产线
    - `铜锭 / 熟铁` 冶铸
    - `精盐 / 药剂 / 麻布 / 皮革` 对人口状态的作用
    - 仓储面板对新材料的展示
    - 制工具优先消耗 `熟铁 / 铜锭`
  - 保留与现有抽象资源兼容映射：
    - `Wood / Stone / ConstructionMaterials`
    - `IndustrialParts / CompositeMaterial`
- 不包含：
  - `T2` 内容：`锡矿 / 青铜 / 竹材 / 石灰石 / 石英砂 / 纸张 / 灰浆 / 漆作`
  - `T3` 内容：`煤 / 焦炭 / 玻璃 / 硫磺 / 硝土 / 火药 / 玉料 / 香木`
  - 地图节点全面改为“只直产自然原材料”的最终态
  - `T0` 建筑链的完整可建造 UI
  - 后期英雄装备专属材料链
  - 商路价格系统重做
  - 完整分支科技树改造

## 实现拆解

1. 资源分层与命名规范落文档
2. 原材料库存字段与兼容迁移
3. 采集 / 初加工 / 民生产线接入小时结算
4. 工具与人口收益挂钩
5. 仓储 UI 展示与构建验证

## 风险与回滚

- 风险：资源条目增多后，仓储 UI 与玩家认知负担上升；若底层字段拆分过快，可能影响旧存档与现有公式；若消费端挂钩不足，可能出现“有材料但无意义”的伪复杂度。
- 回滚方式：保留现有抽象资源汇总字段作为兜底层；若细分链路不稳定，则先退回“表现层细分、底层汇总”的方案；`T2/T3` 全部延后，优先保证 `T0/T1` 的最小闭环稳定。

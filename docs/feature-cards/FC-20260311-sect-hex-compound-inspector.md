# 功能卡（Feature Card）

## 功能信息

- 功能名：天衍峰院域坊局与全格检视（一期）
- 优先级：`P1`
- 目标版本：2026-03-11
- 关联系统：`SectMap`、`TownMapGeneratorSystem`、`CountyTownMapViewSystem`、`Tile Inspector`、`docs/02_system_specs.md`

## 目标与用户收益

- 目标：把天衍峰山门图从“纯地面六角格展示”提升为“每个地块都可检视的院域经营底盘”，为后续同地块多建筑、共享灵气、坊局协同和随机事件留出统一数据骨架。
- 玩家可感知收益（10 分钟内）：点开任意 hex 都能读到中文信息，知道这块地的地势、灵气、可建坊位、天然特征与推荐用途，不再只能把山门图当背景看。

## 实现范围

- 包含：
  - 为天衍峰山门图补一套中文院域坊局设计规格；
  - 定义“固定地块 + 院域坊局 + 共享灵气 + 局部随机性”的一期数据结构；
  - 规定全格点击检视的交互目标、信息层次与动作模板；
  - 将该功能包登记到 `05 / 08` 看板，作为正式开发主线。
- 不包含：
  - 完整的地块内多建筑编辑器；
  - 存档迁移；
  - 随机事件数值调平与完整产线结算接入。

## 实现拆解

1. 在 `docs/02_system_specs.md` 中补“院域坊局”系统规格，定义地块层、坊局层、运行时层和检视层。
2. 在 `docs/08_development_list.md` 中登记正式功能包，明确飞轮定位、依赖与 DoD。
3. 在 `docs/05_feature_inventory.md` 中登记当前状态，避免后续实现时重复插入零散功能。
4. 一期代码落地时，优先做“全格点击检视 + 数据骨架”，暂不直接做地块内拖拽布局。

## 分阶段目标

- 一期：任意 hex 可点击检视，左栏能显示地块底盘、灵气、坊位、天然特征与推荐动作。
- 二期：开放同地块 `2~3` 个建筑槽位，接入共享灵气、协同收益与互扰惩罚。
- 三期：接入节气波动、局部随机事件和驻守弟子差异，避免坊局模板固化。

## 验收标准（可测试）

- [ ] `docs/02_system_specs.md` 已明确写出院域坊局的一期数据结构、交互目标与随机性来源。
- [ ] `docs/05_feature_inventory.md` 与 `docs/08_development_list.md` 已登记该功能包，状态与规格一致。
- [ ] 功能定义明确区分“地块固定底盘”和“地块内多建筑坊局”，避免再次退回 `1 地块 = 1 建筑` 的窄模型。
- [ ] 随机性设计明确为“改题而非掀桌”，不会直接破坏玩家的长期规划感。

## 风险与回滚

- 风险：如果一期把“多建筑 + 灵气共享 + 随机性”一次写得过细，后续实现会被文档反向绑死。
- 回滚方式：保留“全格检视 + 地块固定底盘”定义，暂时下调坊局与随机性细节，只保留字段与原则，不硬写早期公式。

# 功能卡（Feature Card）

## 功能信息

- 功能名：Main 场景 Figma 布局切换开关
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`UI`

## 目标与用户收益

- 目标：在 `Main.tscn` 中挂载 Figma 组合场景，并提供一键回滚到现有布局的安全开关。
- 玩家可感知收益（10 分钟内）：可直接在主场景试跑新布局，同时保持旧布局可立即恢复。

## 实现范围

- 包含：
  - `Main.tscn` 挂载 `scenes/ui/figma/MakeGameUI.tscn`
  - `Main.cs` 新增导出开关，控制显示“旧布局 / Figma 布局”
  - 启动时应用布局模式，默认保留旧布局
- 不包含：
  - 现有 `Main.cs` 数据绑定迁移到 Figma 子场景
  - 玩法、数值、存档结构改动

## 实现拆解

1. 补功能卡并明确回滚策略
2. 在 `Main.tscn` 注入 Figma 组合场景节点
3. 在 `Main.cs` 增加可导出的布局切换开关

## 验收标准（可测试）

- [ ] `Main` 节点有可切换的布局开关（Inspector）
- [ ] 开关关闭时显示旧布局，开启时显示 Figma 布局
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：未来若直接迁移数据绑定，可能出现两套 UI 状态不一致
- 回滚方式：将 `UseFigmaLayout` 设为 `false`，或移除本次新增节点与开关代码

# 功能卡（Feature Card）

## 功能信息

- 功能名：基于中式庭院素材的 UI 主题包
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`UI`

## 目标与用户收益

- 目标：按素材风格生成统一的 Godot UI 主题，覆盖面板、按钮、文本、日志等核心控件。
- 玩家可感知收益（10 分钟内）：界面从默认风格变为统一“青瓦木构”视觉语言，沉浸感更强。

## 实现范围

- 包含：
  - 新增主题资源 `Theme`（配色、按钮状态、面板边框、文本色）
  - 将主题挂为项目全局主题
  - 保持现有 UI 结构和交互逻辑不变
- 不包含：
  - 美术贴图导入与九宫格纹理替换
  - 玩法规则和数值改动
  - 字体资产替换（仍使用当前默认字体）

## 实现拆解

1. 从素材抽取主色（青瓦灰、木棕、竹篱绿、灯笼橙、宣纸米色）
2. 生成 Godot `Theme` 资源并覆盖常用控件样式
3. 在 `project.godot` 配置全局主题并验证构建

## 验收标准（可测试）

- [x] 新主题文件可在工程中加载
- [x] `PanelContainer`、`Button`、`Label`、`RichTextLabel` 样式完成替换
- [x] 主场景运行后整体视觉风格与素材一致性明显提升
- [x] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：主题对比度过低可能影响可读性。
- 回滚方式：移除 `project.godot` 的全局主题引用并回退主题资源文件。

# 功能卡（Feature Card）

## 功能信息

- 功能名：宗主中枢书卷化（治宗册）
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`TaskPanel / MainTaskPanel / SectTaskRules / SectGovernanceRules / SectRuleTreeRules`

## 目标与用户收益

- 目标：将宗主中枢面板统一到“浮云宗书卷子面板”规范，用治宗册替代深色管理弹窗。
- 玩家可感知收益（10 分钟内）：打开宗主中枢时，可在统一的卷轴语境里查看治宗重心、法旨、治理条目与执事落实详情，不再与仓储/留影录/机宜卷风格割裂。

## 实现范围

- 包含：
  - 将 `TaskPanel` 的动态 UI 重构为木轴宣纸式治宗册
  - 统一标题、批注、治理条目、行内切换按钮与底部法令按钮的卷轴语义
  - 保持治理调整、季度法令、门规树与恢复均衡逻辑不变
- 不包含：
  - 不改治理公式、小时结算和底层任务折算逻辑
  - 不新增治理条目或新系统入口

## 实现拆解

1. 重构 `TaskPanel.cs` 的动态布局为卷轴式治宗册
2. 统一 `ItemList / Label / Button` 纸面与墨线样式
3. 将提示与状态反馈改成治宗册语义
4. 同步 `docs/02/05/08` 中宗主中枢描述
5. 执行 `dotnet build .\\Finally.sln`

## 验收标准（可测试）

- [x] 宗主中枢呈现书卷化外观，与仓储账册、留影录、机宜卷风格一致
- [x] 治理条目列表、详情、发展方向/法令/育才/季度法令/门规切换仍可正常交互
- [x] `dotnet build .\\Finally.sln` 通过

## 风险与回滚

- 风险：`TaskPanel` 由代码动态构建，布局改动一旦失衡，可能影响不同窗口尺寸下的按钮与列表可读性。
- 回滚方式：回退 `TaskPanel.cs` 的书卷布局与文案改动，保留治理逻辑和 `MainTaskPanel` 事件链路不变。

## 二轮补强（侧边栏二级菜单重排）

- 背景：原“治宗法旨”单页长列表导致信息层级混杂，策略查找成本偏高。
- 目标：改为“左侧经脉导航 + 右侧卷宗内容”的二级菜单结构，按 `大政方针 / 节气法旨 / 门规戒律 / 庶务调度` 分组。
- 设计约束：
  - 保留“宣纸水墨 + 竹简木刻”表现基调；
  - 行内切换由“前令/后令”文本按钮改为 `◀ 当前态 ▶`；
  - 右侧内容区独立滚动，左侧导航不滚动；
  - 保留既有治理事件链：`Development / Law / Talent / Quarter / RuleTree / TaskOrder`。
- 验收补充：
  - [x] 左侧菜单可点击切换分组，当前分组高亮；
  - [x] 右侧条目按分组展示为卡片，策略切换按钮可直接生效；
  - [x] “庶务调度”仍可查看任务详情并执行 `收敛/推进/鼎力推进/复归常制`；
  - [x] `dotnet build .\\Finally.sln` 通过。

# 功能卡（Feature Card）

## 功能信息

- 功能名：多存档面板筛选排序（V1）
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`SaveSlotsPanel / SaveSystem / MainSaveSlotsPanel`

## 目标与用户收益

- 目标：让玩家在手动槽、主存档、三槽自动存档并存时，能更快定位要覆盖或读取的目标槽位。
- 玩家可感知收益（10 分钟内）：打开多存档面板后，可以按槽位类型筛选、按经营状态排序，减少在列表中逐个比对的时间。

## 实现范围

- 包含：
  - 多槽面板的筛选控件
  - 多槽面板的排序控件
  - 列表计数与空结果提示
- 不包含：
  - 存档截图预览
  - 全文搜索
  - 云存档同步

## 实现拆解

1. 为多槽面板增加筛选/排序控件
2. 将槽位列表切为“全量数据 + 可见列表”双层视图
3. 在切换筛选排序时保持可见槽位选择与详情同步
4. 回写系统规格与开发看板

## 验收标准（可测试）

- [ ] 面板可切换 `全部 / 主存档 / 手动槽 / 自动槽`
- [ ] 面板可按最近写入、游戏进度、人口、金钱、科技排序
- [ ] 切换筛选排序后列表与详情同步更新且不报错
- [ ] 空筛选结果时有明确提示，不出现失效按钮误触发
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：筛选项过多会压缩现有面板空间；列表切换若没有同步选择，可能出现详情与所见不一致。
- 回滚方式：保留面板布局与底层槽位逻辑，只回退筛选排序控件与可见列表逻辑。

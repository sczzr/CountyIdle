# 功能卡：主界面六边形沙盘重构

## 功能信息

- 功能名：主界面六边形沙盘重构
- 优先级：`P1`
- 目标版本：当前迭代（2026-03-09）
- 关联系统：`CountyIdle/scenes/Main.tscn`、`CountyIdle/scenes/ui/WorldPanel.tscn`、`CountyIdle/scenes/ui/JobsPanel.tscn`、`CountyIdle/scenes/ui/TopBar.tscn`、`CountyIdle/scenes/ui/BottomBar.tscn`、`CountyIdle/scenes/ui/EventLogPanel.tscn`、`CountyIdle/scripts/systems/CountyTownMapViewSystem.cs`

## 目标与用户收益

- 目标：将主界面重心重新收敛到“中央六边形沙盘”，让玩家围绕地块选中、检视与下达指令展开操作。
- 玩家可感知收益（10 分钟内）：进入主界面后，中央地图面积更大、左右信息更聚焦，点击天衍峰场所时能在左侧直接看到地块状态与快捷操作。

## 实现范围

- 包含：
  - `Main.tscn` 改为“中央沙盘 + 左右浮动面板 + 底部控制台”布局；
  - `WorldPanel.tscn` 的地图页签改为底部悬浮控制台样式；
  - 左侧新增地块检视卡，接收天衍峰山门图的场所选中摘要；
  - 顶栏 / 底栏 / 日志面板切到更轻的琉璃化视觉风格。
- 不包含：
  - 不重写天衍峰 hex 地图生成算法；
  - 不新增新的地图类型或第三张主地图；
  - 不改小时结算、人口、资源、战斗等核心数值逻辑。

## 实现拆解

1. 调整 `Main.tscn` 与相关 UI 场景，把中央空间优先让给沙盘地图。
2. 为天衍峰山门图补一条“场所选中摘要 -> 左侧检视卡”的 UI 数据通路。
3. 保持原有地图页签、宗主中枢、弟子谱、仓储入口可用，但重新组织位置与层级。
4. 回填功能看板、开发列表与系统规格。

## 验收标准（可测试）

- [x] 左键选中天衍峰场所后，左侧检视卡会刷新标题、状态、值守 / 路上门人和 Hex 坐标。
- [x] 地图页签、宗主中枢、弟子谱、仓储、存档/读档、倍速按钮仍可找到并继续工作。

## 风险与回滚

- 风险：本次主要改的是场景层级与视觉结构，若节点路径漂移，`Main.cs` 的 UI 绑定容易失效。
- 回滚方式：恢复 `Main.tscn / WorldPanel.tscn / JobsPanel.tscn / TopBar.tscn / BottomBar.tscn / EventLogPanel.tscn` 到上一版布局，并移除地块检视摘要绑定。

## 二轮补强

- 继续压窄左右浮动面板，让中央沙盘视区更接近“画框式”体验。
- 将地图调度条从上沿收到底部页签上方，减少对首屏地图的遮挡。
- 将世界图按钮统一为 `世界大地图`，与当前主界面叙事一致。

## 三轮补强

- 将 `TopBar` 进一步压薄，强化“顶部天幕”感。
- 继续上移中央地图和左右浮动栏，扩大首屏沙盘可视区域。

## 四轮补强

- 将 `仓储 / 宗主中枢 / 弟子谱` 从地图页签行收到底部控制台。
- 顶部地图控件区只保留地图切换与缩放，继续减少中心画面周边按钮密度。

## 五轮补强

- 将左侧检视器明确收口为 `Tile Inspector` 卡面结构。
- 将默认文案和选中态文案改得更像“地块检视卡”，减少技术字段直出感。

## 六轮补强

- 将右侧 `宗门纪事` 面板进一步收成“微缩日志窗”，降低边框和警示块的视觉重量。
- 保持 `LogLabel` 节点路径不变，只微调 `EventLogPanel.tscn` 的层级样式与静态示例文案，避免影响 `Main.cs` 现有绑定。

## 七轮补强

- 新增独立摘要规则与 UI 绑定文件，继续避免把右栏判断逻辑堆进 `Main.cs`。

## 八轮补强

- 将左侧 `Tile Inspector` 的三级动作改为更具体的 tile 专属操作项，不再长期停留在通用按钮文案。
- 未选中任何 tile 时，动作按钮默认禁用并提示先点选中央 hex tile；选中后再按地块类型切换到扩建 / 调度 / 打开对应面板。

- 选中不同 tile 时同步切换强调色与状态色，让灵田、工坊、总坊、传法院、庶务殿、休憩点在左侧卡面一眼区分。

## 十轮补强

- 抽出共享的 `tile 类型配色规则`，让左侧检视卡和中央地图的选中高亮不再各自为政。
- 中央选中 tile 新增双层 halo、外描边与入口路径高亮，让“地图上选中了哪一块”更直观。

## 十一轮补强

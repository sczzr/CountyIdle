# 功能卡：世界格二级地图分层与入口系统

## 功能信息

- 功能名：世界格二级地图分层与入口系统
- 优先级：`P1`
- 目标版本：地图空间分层一期
- 关联系统：`世界地图`、`二级地图模板`、`世界生成标签`、`地图入口/退出规则`

## 目标与用户收益

- 目标：让玩家在世界地图点击不同格子时，进入具有明确身份与玩法的二级地图，而不是统一落到同一种视图。
- 玩家可感知收益（10 分钟内）：世界地图不再只是选点看信息，玩家能明显感知“这个格子为什么值得进去”，并理解不同地点如何服务宗门发展、历练、贸易和外域协同。

## 实现范围

- 包含：
  - 定义世界格到二级地图的映射规则；
  - 首批讨论七类二级地图：`Sect / Wilderness / MortalRealm / CultivatorClan / ImmortalCity / Market / Ruin`，对应 `宗门 / 野外 / 凡俗国度 / 修仙世家 / 仙城 / 坊市 / 遗迹`；
  - 为每类地图定义主玩法、风险、产出、回流宗门的方式；
  - 约束进入条件、退出规则与时间推进方式；
  - 输出世界生成规则草案，包括区块、权重、邻接、稀有度与解锁层级。
  - 输出接近配置表的参数草案，供后续 `JSON` 与 `C#` 模型直接采用。
- 不包含：
  - 一次性做完所有二级地图的完整可玩版本；
  - 立即重写现有天衍峰山门图与世界图渲染框架；
  - 在本卡阶段直接敲定所有细分建筑、事件和掉落表。

## 实现拆解

1. 先定义世界格元数据：地块类型、阵营归属、风险、资源主轴、推荐交互。
2. 再定义二级地图模板：据点经营型、势力关系型、野外历练型、高风险副本型。
3. 再补世界生成规则：大区块、选点优先级、邻接加权、稀有度与门槛。
4. 再把生成规则收成参数草案：字段、推荐权重区间、伴生概率与门槛。
5. 最后确定点击进入、退出回流、时间流逝和与现有飞轮系统的对接顺序。

## 验收标准（可测试）

- [ ] 世界格与二级地图类型之间的映射规则可列成清单，且不会出现“看起来不同、进去却完全一样”的空映射。
- [ ] 七类二级地图都能说清自己的玩家目标、核心操作、产出与风险。
- [ ] 文档能明确指出哪些类型先做、哪些后做，以及为什么这样排序。
- [ ] `仙城` 与 `坊市`、`凡俗国度` 与 `修仙世家`、`野外` 与 `遗迹` 三组边界在规格中被明确写死，避免实现重叠。
- [ ] 世界生成规则能回答“为什么这个点会长在这里”，并可拆成区块、权重、邻接、稀有度和解锁五层配置。
- [ ] 参数草案已能直接映射到 `JSON` 和 `C#` 模型字段，不需要再发明第二套命名。

## 风险与回滚

- 风险：
  - 如果二级地图按地名硬拆，容易做成大量皮肤不同但玩法重复的空壳地图；
  - 如果每类地图都自带独立规则，容易把主循环切碎，增加维护成本；
  - 如果进入地图没有清晰回流，玩家会觉得在“逛景点”，而不是在经营宗门。
- 回滚方式：
  - 保留统一的“地图类型模板”中间层，若细分类太多，可先收敛为少量模板，再继续复用世界观包装。

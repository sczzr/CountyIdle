# 功能卡（Feature Card）

## 功能信息

- 功能名：县城地图程序化生成（Watabou 风格参考 + 中式 2.5D 视角）
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`地图生成 / 县城可视化`

## 目标与用户收益

- 目标：在“县城地图”页签提供可程序化生成的街道+房屋布局，并采用 2.5D 斜面渲染，替代纯文本占位。
- 玩家可感知收益（10 分钟内）：可以直观看到带道路肌理的中式城镇轮廓，地图层级切换更有反馈感。

## 实现范围

- 包含：
  - 新增县城地图生成系统（道路骨架、地块分配、房屋布置）
  - 新增 2.5D 县城绘制脚本（地块、道路、中式屋顶/墙面）
  - 县城地图页支持“一键重生成”以刷新布局
  - `Main.cs` 按状态同步人口提示，驱动地图密度变化
- 不包含：
  - 世界地图/郡地图的真实地理生成
  - 寻路、建筑交互、点击选中等玩法逻辑
  - 地图数据写入存档格式

## 实现拆解

1. 定义 TownMap 数据模型与生成规则（道路→地块→房屋）
2. 实现县城地图视图脚本并完成 2.5D 渲染
3. 挂接 `WorldPanel` 县城页签与 `Main.cs` 状态同步入口

## 验收标准（可测试）

- [ ] 县城地图页签显示程序化城镇，而非文本占位
- [ ] 房屋为中式配色风格，并以 2.5D 斜面方式绘制
- [ ] 点击“重生成”可刷新街道与房屋布局
- [ ] 人口变化后，房屋数量会出现可感知变化
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：地图绘制复杂度提升后，低端设备 UI 刷新可能出现抖动。
- 回滚方式：回退 `WorldPanel.tscn`、`Main.cs` 与新增地图脚本文件，恢复文本占位版本。

# 功能卡（Feature Card）

## 功能信息

- 功能名：自动存档槽策略（V1：单自动槽）
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`SaveSystem / SqliteSaveRepository / MainUI / GameLoop / SaveSlotsPanel`

## 目标与用户收益

- 目标：在 SQLite 多槽体系上补充单自动存档槽 `autosave`，按固定小时结算间隔自动写入，降低误操作和突发退出带来的回档损失。
- 玩家可感知收益（10 分钟内）：即使没有手动存档，运行一段时间后也会自动出现“自动存档”槽，可直接在多槽面板里读取。

## 实现范围

- 包含：
  - 新增自动存档槽 `autosave / 自动存档`
  - 每 `6` 次小时结算自动写入一次自动存档
  - 多槽面板展示自动存档槽
  - 自动存档槽禁止手动覆盖、重命名、删除
  - 快速存档 / 快速读档继续只操作主存档
- 不包含：
  - 自动存档轮换槽
  - 自动存档保留策略配置
  - 存档差分写入
  - 退出游戏时额外自动存档

## 实现拆解

1. 扩展 `SaveSystem` 增加自动存档槽常量与写入方法
2. 在主循环状态刷新链路中接入自动存档触发判断
3. 在多槽面板中标记并保护自动存档槽
4. 回写系统规格与开发列表

## 验收标准（可测试）

- [ ] 当小时结算次数跨过 `6` 的倍数时，会自动写入 `autosave` 槽
- [ ] 多槽面板中可以看到 `自动存档` 槽
- [ ] 自动存档槽可以读取，但不能重命名、删除、手动覆盖
- [ ] 快速存档 / 快速读档仍然走主存档 `default`
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：若自动存档过于频繁，会产生额外 IO；若触发条件写错，可能在读档后立刻覆盖自动槽。
- 回滚方式：保留多槽 UI，仅关闭自动存档触发逻辑与 `autosave` 槽写入。

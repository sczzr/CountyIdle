# 功能卡（Feature Card）

## 功能信息

- 功能名：双地图兼容页签入口收口
- 优先级：`P1`
- 目标版本：`当前迭代`
- 关联系统：`Main`、`WorldPanel`

## 目标与用户收益

- 目标：让主界面地图页签入口与当前 `DL-027` 双地图布局裁定完全一致，避免历史兼容页签继续保留“看似可参与主链、实际被回退”的误导代码。
- 玩家可感知收益（10 分钟内）：主界面的地图入口继续稳定聚焦在 `山门沙盘 / 世界舆图` 两条主链，旧 `Prefecture / Event / Report / Expedition` 不会再作为现行入口链的一部分混入点击逻辑。

## 实现范围

- 包含：
  - 将历史兼容地图页签从现行点击绑定与主链必需节点中剥离
  - 保持兼容节点隐藏且禁用，继续允许场景内部保留节点
  - 回写 `docs/02_system_specs.md`、`docs/05_feature_inventory.md`、`docs/08_development_list.md`
- 不包含：
  - 不恢复 `Prefecture / Event / Report / Expedition` 页面
  - 不删除这些兼容节点或改动其底层视图容器
  - 不改世界图、山门图、二级地图与地图渲染 authority

## 实现拆解

1. 复核 `DL-027` 当前文档裁定与 `Main.cs / WorldPanel.tscn` 现状
2. 将历史兼容页签从现行主界面点击绑定和必需节点校验中移出
3. 保持兼容按钮隐藏且禁用，并执行场景烟测与构建验证

## 验收标准（可测试）

- [ ] 主界面地图点击链只绑定 `山门沙盘 / 世界舆图`
- [ ] `Prefecture / Event / Report / Expedition` 兼容按钮保持隐藏且禁用
- [ ] `SetMapTab()` 不再把这些兼容按钮和视图当作双地图主链的必需节点
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：若误把仍有兼容用途的节点完全移除，可能影响历史 fallback 视图或未来恢复计划
- 回滚方式：恢复 `Main.cs` 中兼容页签的旧绑定/校验逻辑，并回退本卡对应文档记录

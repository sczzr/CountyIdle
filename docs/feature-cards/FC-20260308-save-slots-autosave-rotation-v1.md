# 功能卡（Feature Card）

## 功能信息

- 功能名：自动存档轮换槽（V1：保留最近 3 个自动档）
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`SaveSystem / SqliteSaveRepository / MainSaveSlotsPanel / SaveSlotsPanel / GameLoop`

## 目标与用户收益

- 目标：将当前“单自动存档槽”升级为“3 槽轮换自动存档”，保留最近三次自动存档落点，降低误覆盖带来的回档损失。
- 玩家可感知收益（10 分钟内）：多存档面板会出现 `自动存档 1 / 2 / 3` 三个受保护槽位，能够回退到更早一轮的自动记录。

## 实现范围

- 包含：
  - 自动存档槽扩展为 3 个：
    - `autosave`
    - `autosave_2`
    - `autosave_3`
  - 每 `6` 次小时结算触发一次自动存档
  - 按轮换规则依次覆盖三个自动存档槽
  - 多存档面板继续保护自动槽，禁止手动覆盖、重命名、删除
- 不包含：
  - 自动存档槽数量自定义
  - 自动存档截图预览
  - 自动存档时间轴筛选

## 实现拆解

1. 扩展 `SaveSystem` 的自动槽常量与保护逻辑
2. 将自动存档写入改为 3 槽轮换
3. 调整多槽面板展示与保护判断
4. 回写系统规格、看板与开发列表

## 验收标准（可测试）

- [ ] 自动存档槽由单槽扩展为 `自动存档 1 / 2 / 3`
- [ ] 每跨过 `6` 次小时结算时，自动存档写入下一个轮换槽
- [ ] 多槽面板中三个位于自动存档线的槽都被标记为受保护槽
- [ ] 自动存档槽依旧可读，但不可手动覆盖、重命名、删除
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：轮换规则若写错，可能导致自动槽顺序混乱或重复覆盖同一槽。
- 回滚方式：退回单自动存档槽实现，保留多槽 UI 与默认槽逻辑。

# 功能卡（Feature Card）

## 功能信息

- 功能名：时间顶栏季度/天双进度条
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`UI / Main / GameCalendarSystem`

## 目标与用户收益

- 目标：把现有单一节气进度条补强为“季度进度 + 当日进度”双层时间反馈，让时间推进更直观。
- 玩家可感知收益（10 分钟内）：能同时看懂“这个季度过了多少”和“今天走到什么时辰”，挂机与切速时更容易把握节奏。

## 实现范围

- 包含：
  - `GameCalendarSystem` 输出季度/天双进度百分比与说明文本
  - Legacy 顶栏新增季度/天两条进度条
  - Figma 顶栏同步新增季度/天两条进度条
  - 进度条随 `GameMinutes` 增长实时刷新，兼容 `x1/x2/x4` 与读档
- 不包含：
  - 不改变 `1 秒 = 1 游戏分钟`
  - 不改变 `60 游戏分钟 = 1 次小时结算`
  - 不新增新的结算资源或季节事件公式

## 实现拆解

1. 扩展日历描述模型，补充季度进度、当日进度与对应文案。
2. 改造 Legacy/Figma 顶栏节点结构，放置两条时间进度条。
3. 在 `Main` 刷新逻辑中同步双进度条与提示文本，并验证倍率/读档兼容。

## 验收标准（可测试）

- [ ] 顶栏出现“季度进度条 + 当日进度条”两条时间条
- [ ] 时间推进时，两条进度条都会随 `GameMinutes` 连续变化
- [ ] Legacy 与 Figma 两套布局都能正常显示双进度条
- [ ] `x1/x2/x4` 与读档后进度条相位保持连续
- [ ] `dotnet build .\\Finally.sln` 通过

## 风险与回滚

- 风险：顶栏高度增加后可能压缩资源栏布局，或读档后双进度条与当前时间不同步。
- 回滚方式：回退 `GameCalendarSystem`、`Main.cs` 与两套顶栏场景，恢复单一节气进度条显示。

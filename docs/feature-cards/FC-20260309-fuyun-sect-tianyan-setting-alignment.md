# 功能卡：浮云宗·天衍峰设定对齐

## 功能信息

- 功能名：浮云宗·天衍峰设定对齐
- 优先级：`P1`
- 目标版本：当前迭代（2026-03-09）
- 关联系统：`docs/09_xianxia_sect_setting.md`、`docs/01_game_design_guide.md`、`docs/02_system_specs.md`、`docs/05_feature_inventory.md`、`docs/08_development_list.md`、`CountyIdle/scripts/systems/SectMapSemanticRules.cs`、`CountyIdle/scripts/systems/SectTaskRules.cs`、`CountyIdle/scripts/ui/TaskPanel.cs`

## 目标与用户收益

- 目标：把当前“泛宗门”背景进一步定锚到参考设定中的 **浮云宗（青云州江陵府）+ 青云峰三总殿 + 天衍峰阵堂** 结构。
- 玩家可感知收益（10 分钟内）：打开文档、任务面板、岗位摘要与地图场所时，能直接读到 `浮云宗 / 天衍峰 / 阵堂 / 庶务殿 / 传法院 / 青云总坊` 这一套有组织来源的名称，而不是抽象占位宗门词。

## 实现范围

- 包含：
  - 更新设定总表、总纲、系统规格、看板与开发列表；
  - 将任务法旨、建筑命名、地图场所与弹窗标题对齐到浮云宗天衍峰语义；
  - 新增本轮归档记录。
- 不包含：
  - 不改变现有数值公式；
  - 不重写世界生成、地图布局和存档结构；
  - 不实现九峰全量玩法差异。

## 实现拆解

1. 抽取参考文档中的可复用设定：宗门名、所在地、三总殿、天衍峰定位。
2. 将核心文档中的背景基线与术语映射改成浮云宗天衍峰版本。
3. 将玩家可见设施、任务与场所名称改成天衍峰语义。
4. 回写看板、开发列表与本轮归档。

## 验收标准（可测试）

- [x] 核心文档明确写出“浮云宗（青云州江陵府）+ 天衍峰 + 青云峰三总殿”。
- [x] 任务法旨与建筑名称改成 `阵材采炼 / 阵枢营造 / 阵法推演 / 青云总坊 / 庶务殿 / 传法院` 等设定化词汇。
- [x] `dotnet build .\Finally.sln` 通过。

## 风险与回滚

- 风险：过度贴近设定后，后续若要切换到别的宗门模板，文本替换点会增多。
- 回滚方式：保留文档中的“默认宗门原型”结构，回退玩家可见命名即可。

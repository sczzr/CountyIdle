# 功能卡（Feature Card）

## 功能信息

- 功能名：多存档槽面板书卷化（留影录）
- 优先级：`P1`
- 目标版本：`v0.x`
- 关联系统：`SaveSlotsPanel / MainSaveSlotsPanel / PopupPanelBase / SaveSystem`

## 目标与用户收益

- 目标：将多存档槽面板统一到“浮云宗书卷子面板”规范，用留影录卷册替代现代弹窗管理面板。
- 玩家可感知收益（10 分钟内）：打开存档面板后，可在统一的卷轴语境里查看卷册目录、详情留影与存档批令，整体与仓储账册风格一致。

## 实现范围

- 包含：
  - 为 `SaveSlotsPanel` 增加木轴、宣纸、金边装裱等书卷式外观
  - 将标题、提示、按钮文案与状态提示切到“卷册 / 留影录”语义
  - 统一按钮、筛选框、列表、输入框与预览框的宣纸/墨线样式
  - 保持多槽筛选、排序、读取、覆盖、新建、复制、重命名、删除逻辑不变
- 不包含：
  - 不改 SQLite 存档结构、自动存档轮换和截图保存逻辑
  - 不新增导入导出或云存档
  - 不改主界面存档按钮入口逻辑

## 实现拆解

1. 重绘 `SaveSlotsPanel.tscn` 的卷轴装饰与文案
2. 在 `SaveSlotsPanel.cs` 中追加宣纸/墨线主题样式
3. 将提示语与交互反馈改成卷册语义
4. 同步 `docs/02/05/08` 对存档面板的描述
5. 执行 `dotnet build .\\Finally.sln`

## 验收标准（可测试）

- [ ] 多存档槽面板呈现书卷化外观，和仓储账册风格一致
- [ ] 卷册目录、详情留影、筛选排序、输入框和操作按钮都能正常显示与交互
- [ ] 读取、覆盖、新建、复制、重命名、删除逻辑保持原有可用性
- [ ] 文案统一使用“卷册 / 留影录 / 落卷”等语义
- [ ] `dotnet build .\\Finally.sln` 通过

## 风险与回滚

- 风险：`ItemList`、`OptionButton` 等 Godot 原生控件的主题覆盖项较多，需要确认不同状态下的可读性稳定。
- 回滚方式：回退 `SaveSlotsPanel.tscn` 与 `SaveSlotsPanel.cs` 的视觉与文案改动，保留多槽存档逻辑与入口不变。

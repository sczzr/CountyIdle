# 功能卡：历史归档语义桥接（V1）

## 功能信息

- 功能名：历史归档语义桥接（V1）
- 优先级：`P2`
- 目标版本：当前迭代（2026-03-08）
- 关联系统：`docs/feature-cards/*`、`docs/change-proposals/*`、`docs/balance-logs/*`

## 目标与用户收益

- 目标：让旧阶段归档在保留历史可追溯性的同时，能被当前“天衍峰经营”背景直接阅读。
- 玩家可感知收益（10 分钟内）：打开旧 `FC / CP / BL` 时，能立即知道这些“县城 / 郡图 / 郡县”词在当前项目里对应什么，不需要自己手动转换语义。

## 实现范围

- 包含：
  - 为旧归档批量补“历史兼容说明”；
  - 为高频旧链路补“当前对外语义”头注；
  - 改写代表性旧归档的标题 / 目标 / 摘要层文案；
  - 将 `countytown-* / prefecture-*` 旧归档的全部正文持续更新到当前宗门 / 外域语义，包括章节标题、实现范围、拆解步骤、验收项与结果描述。
- 不包含：
  - 不重命名历史 `canonical-topic`；
  - 不整批重写所有旧正文细节；
  - 不修改代码层历史类名。

## 实现拆解

1. 盘点高频旧语义归档链路。
2. 统一补“历史兼容说明”头注。
3. 为关键链路补“当前对外语义”与宗门化摘要。
4. 回写注册表、索引与开发列表。

## 验收标准（可测试）

- [x] 旧归档打开后可直接读到“历史兼容说明”。
- [x] 高价值旧链路新增“当前对外语义”头注。
- [x] `countytown-* / prefecture-*` 目标旧归档 `41` 份全部补齐“当前对外语义”头注。
- [x] `countytown-* / prefecture-*` 旧归档的功能名 / 标题 / 改动项 / 实现范围 / 拆解步骤 / 验收项 / 结果描述已完成宗门化收口。
- [x] `dotnet build .\Finally.sln` 通过。

## 风险与回滚

- 风险：旧归档正文深处仍保留大量历史阶段细节，若继续深挖阅读，仍会碰到旧语义。
- 回滚方式：保留索引和注册表更新，仅移除新增的头注与摘要层改写即可。

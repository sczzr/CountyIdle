# 功能卡（Feature Card）

## 功能信息

- 功能名：产业链岗位逻辑重构（建筑+工具约束）
- 优先级：`P0`
- 目标版本：`v0.x`
- 关联系统：`GameState / IndustrySystem / EconomySystem / GameLoop / JobsPanel`

## 目标与用户收益

- 目标：建立“产业出现 → 建筑建设 → 工具供给 → 岗位产出”的生产关系闭环。
- 玩家可感知收益（10 分钟内）：岗位加点会受建筑容量与工具覆盖影响，并在日志看到建造/缺口反馈。

## 实现范围

- 包含：
  - 四岗位语义调整为：产业工人、管理人员、商业人员、研发人员
  - 新增建筑与工具状态字段并可存档
  - 新增产业系统：岗位容量约束、手动建造、工具产耗
  - 经济系统按工具覆盖和管理覆盖结算产出
  - 左侧岗位面板改为显示岗位新语义和容量状态
  - 底栏新增产业操作按钮（建造五类建筑 + 制工具）
- 不包含：
  - 建筑手动建造 UI
  - 多级产业链（原料 → 半成品 → 成品）
  - 独立建筑面板与图鉴系统

## 实现拆解

1. 更新系统规格并补改动提案
2. 新增产业规则与产业系统，接入小时结算
3. 调整岗位分配约束与经济结算公式
4. 更新岗位面板文案与状态展示
5. 构建验证并记录平衡日志

## 验收标准（可测试）

- [ ] 无对应建筑时，岗位不可超出容量上限
- [ ] 工具不足时，产出按覆盖率衰减且无负资源
- [ ] 管理岗位提升后，经济产出出现可见增益
- [ ] 手动建造/制工具按钮可触发且日志反馈清晰
- [ ] 旧存档可读档并继续结算
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：新增约束后早期资源可能偏紧，玩家短期挫败感上升。
- 回滚方式：停用 `IndustrySystem`，恢复经济系统对岗位的直接线性产出。

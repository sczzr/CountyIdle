# 功能卡（Feature Card）

> 历史兼容说明（2026-03-08）：本文保留旧阶段的“县城 / 郡图 / 郡县 / 州府”等表述以便追溯；在当前项目语义中，请分别按“天衍峰驻地 / 江陵府外域 / 天衍峰经营 / 世界地图”理解。
> 当前对外语义：天衍峰山门图实体场所 / 弟子活动锚点


## 功能信息

- 功能名：宗门弟子可视移动系统（四期：实体场所）
- 优先级：`P2`
- 目标版本：`v0.x`
- 关联系统：`TownMapGeneratorSystem / CountyTownMapViewSystem / TownMapData`

## 目标与用户收益

- 目标：让弟子“去哪里上工、去哪里休闲”在天衍峰山门图上有真实建筑承载，而不是只有抽象锚点。
- 玩家可感知收益（10 分钟内）：打开天衍峰山门图即可分辨阵材圃作业点、傀儡工坊、山门坊市、传法院、庶务殿、论道亭等实体场所，弟子行为更可解释。

## 实现范围

- 包含：
  - 为 `TownActivityAnchorData` 增加实体场所建筑字段（地块、朝向、层数、变体）
  - 在地图生成阶段为每类场所选取临路空地作为 `LotCell`
  - 将场所从圆点标记升级为 2.5D 小建筑渲染
  - 保持弟子仍以临路道路点作为到岗与休闲停留位
- 不包含：
  - 场所点击交互与详情面板
  - 场所占用/容量对经营结算的反向影响
  - 独立场所资源生产链或工作槽位系统

## 实现拆解

1. 更新 `docs/02_system_specs.md` 冻结场所实体字段与选址边界
2. 扩展 `TownActivityAnchorData`，补齐实体建筑参数
3. 在 `TownMapGeneratorSystem` 中选择不与住宅冲突的场所地块
4. 在 `CountyTownMapViewSystem` 中按深度统一绘制住宅与场所实体
5. 编译验证并补交付日志

## 验收标准（可测试）

- [ ] 每类场所锚点都拥有实际 `LotCell`，且不占用道路与水域
- [ ] 天衍峰山门图上的场所显示为可辨识的小型实体建筑，而不是圆点
- [ ] 弟子作息与通勤逻辑保持不变，仍只影响视觉层
- [ ] 住宅与场所前后遮挡关系稳定，无明显绘制穿插错误
- [ ] `dotnet build .\Finally.sln` 通过

## 风险与回滚

- 风险：小地图局部过密时，场所地块选择可能失败或与住宅视觉上过于拥挤。
- 回滚方式：保留锚点数据结构与弟子作息逻辑，仅将场所渲染回退为简化标记。

# CountyIdle 修仙弟子修为、功法、技艺、身份整理（与现行系统对齐）

> 本文是 CountyIdle 的专题设计文档，用于把“修仙弟子修为 / 功法 / 技艺 / 身份”的框架与现行游戏系统对齐。  
> 它只回答三件事：  
> 1) 现行系统里“谁在做事、能做什么、能做到多深”的统一表达  
> 2) 修为/功法/技艺/身份如何支撑宗门飞轮，而不是漂浮名词表  
> 3) 后续系统（传承分支、英雄、装备、Boss）如何接入同一套语义  
>
> 规则与数值边界以 `docs/02_system_specs.md` 为准；世界观与术语以 `docs/09_xianxia_sect_setting.md` 为准。

## 1) 设计目标

- 把“弟子是谁 / 修到哪 / 学什么 / 担任什么”落成四个稳定的玩家认知点。
- 让弟子调度、传承研修、外务历练、装备打造等系统共用一套语义。
- 避免“职业歧视”叙事，把“身份/职司”当成分工与承载，而非价值高低。

## 2) 设计边界（不做什么）

- 不在本文定义数值公式、收益倍率、结算顺序。
- 不要求玩家微观操控每个弟子的修炼细节。
- 不把“修仙名词”当皮肤，必须能落回飞轮与决策。

## 3) 核心模型：四要素标签

修仙弟子在系统中用“四要素标签”描述，避免无限新增概念：

| 维度 | 回答的问题 | 系统作用 |
| --- | --- | --- |
| 修为（境界） | 能承受多强的任务与风险 | 决定风险上限、外务层级与关键职司门槛 |
| 功法（法门/传承路线） | 以何种法门修炼、走哪条传承路径 | 解锁/约束技艺分支，塑形成长倾向 |
| 技艺（术门） | 擅长做什么、产出什么 | 决定任务风格、产出方向与战斗表现 |
| 身份（归属/职司） | 在宗门中担任什么位置 | 决定职责、权限与调度范围 |

关系要点：

- 修为给“上限”，功法给“路径”，技艺给“产出/打法”，身份给“职责”。
- 功法 ≠ 技艺：功法是修行路线与法门，技艺是可执行的能力方向。
- 同一修为下可有不同功法与技艺组合，避免“境界=一切”。

## 4) 身份分类（身份轴）

| 分类 | 说明 | 主要对接系统/场景 |
| --- | --- | --- |
| 门人 | 宗门人口池的主体，承担基础职司与日常运转 | `Population`、`Jobs`、宗门日常 |
| 执事 | 具备管理权或关键职司资格的门人 | 宗主中枢、宗务任务 |
| 真传/核心战力 | 可独立完成高风险/高收益事务的单位 | 外务历练、护山战备（后续英雄实体） |
| 客卿/供奉 | 以契约或关系加入宗门的高阶外力 | 事件、交易、外交与突发支援 |
| 外域修士（散修/世家/隐修） | 宗门之外的行动方 | 外域事件、商路、威胁与机缘 |

> 注：以上“外域修士”只代表立场与组织归属，不暗含价值判断。

## 5) 修为境界（修为轴）

修为层级用于控制任务边界、风险上限与成长阶段，避免“等级=一切”。

| 境界 | 系统含义（不含数值） | 主要作用 |
| --- | --- | --- |
| 炼气 | 进入修行序列，能承担基础职司 | 日常产线、基础任务 |
| 筑基 | 稳定修行，能承担关键岗位 | 关键职司、宗门稳定 |
| 金丹 | 可担任核心职司与外务骨干 | 外务历练、重大任务 |
| 元婴 | 具备独立行动与指挥价值 | 事件主力、护山核心 |
| 化神及以上 | 代表宗门威望与战略上限 | 高阶历练、Boss机制、宗门威慑 |

> 境界不直接替代“技艺”，而是决定技艺能发挥到哪一层。

## 6) 功法谱系（功法轴）

功法是“传承路线 + 法门标签”，描述弟子走哪条修行路径，直接影响技艺分支的解锁与成长倾向。

| 层级 | 系统含义 | 主要作用 |
| --- | --- | --- |
| 入门功法 | 门内基础修行路线 | 允许学习基础技艺，定义初始倾向 |
| 主修法门 | 主要修行路径 | 影响技艺优先级与成长走向 |
| 真传心法 | 高阶核心传承 | 解锁高阶技艺与关键战备表现 |

> 当前阶段允许功法先以“标签/法门名”形式存在，数值规则归 `02` 定义。

## 7) 技艺谱系（技艺轴）

技艺按宗门飞轮拆成三类群，保证“产出—传承—战备”三线闭环。

| 技艺 | 核心价值 | 对接系统 |
| --- | --- | --- |
| 炼丹 | 保障门人恢复、突破与长期稳定 | 人口恢复、任务续航 |
| 炼器 | 打造装备/法器，放大外务收益 | 装备打造、历练收益 |
| 阵法 | 宗门防御与局部增益 | 护山战备、宗门设施 |
| 符箓 | 临战爆发与任务效率提升 | 外务历练、事件结算 |
| 剑道/体修 | 直接战斗强度的基线 | 外务历练、Boss机制 |
| 御兽 | 扩展行动范围与行动效率 | 外务历练、资源回收 |
| 灵植 | 稳定资源链与炼丹支撑 | 资源系统、仓储 |
| 卜算 | 事件预警、风险控制 | 事件系统、决策提示 |
| 医道 | 损耗控制与危机缓解 | 人口系统、恢复链 |

> 技艺是“擅长方向”，不是职业高低；同一修为可有不同技艺路径。

## 8) 与现行系统的映射

| 现行系统 | 术语映射 | 本文增量含义 |
| --- | --- | --- |
| `Population` | 门人总量 | 引入身份轴的语义层 |
| `Jobs` | 职司 | 职司由“身份 + 修为 + 功法 + 技艺”共同决定范围 |
| `Research` | 传承研修 / 功法推演 | 解锁功法路线与高阶法门 |
| `Breeding` | 灵根苗子培养 | 影响修为成长潜力与技艺倾向 |
| `Combat` | 外务历练 / 护山战备 | 修为决定风险层级，功法与技艺决定战斗风格 |
| `Equipment` | 装备 / 法器 | 炼器与阵法成为核心输入端 |

## 9) 飞轮中的作用

- 人口繁衍形成“门人池”，为身份与职司提供基础盘面。
- 修为提升解锁更高阶任务与关键职司。
- 传承研修推动功法分支，带来技艺分化与路线差异。
- 技艺产出反哺资源链与外务收益。
- 真传/核心战力实体化后进入高风险历练与护山战备。
- 装备与法器把外务收益转化为宗门长期优势。

## 10) UI 与反馈要点（原则级）

- 门人信息至少能看出：修为、功法、技艺、身份。
- 概览展示使用三段式修为：`初阶(炼气/筑基)`、`中阶(金丹/元婴)`、`高阶(化神+)`。
- 详细展示保留完整境界名，并允许补充次要技艺标签。
- 功法优先显示“主修法门”，若未生成显示“待定功法”。
- 技艺显示优先遵循“主修优先、上下文优先”，缺失则显示“待定技艺”。
- 禁用“等级/职业高低”叙述，统一用“修为/功法/技艺/身份”。

## 11) 伦理与叙事约束

- 身份与职司表达“分工与承载”，不表达阶级优劣。
- 对外冲突叙事聚焦“守护、代价、秩序”，避免去人性化表述。

## 12) 已确认口径与待确认

已确认口径（与 `docs/02_system_specs.md` 对齐）：

- 功法命名采用“传承路线关键词 + 法门名”，优先使用宗门/峰堂/道统语义，避免现代科技或职业称谓。
- 功法允许分层：入门功法 -> 主修法门 -> 真传心法（语义层，不强行数值）。
- 研修系统解锁功法路线节点；节点可授予技艺标签或开启高阶职司门槛。
- 技艺由“功法路线 + 修为门槛”共同解锁，避免单一研究点直接授予战力。
- UI 概览顺序固定为：`身份 -> 修为 -> 功法 -> 技艺`。

待确认（仍保留讨论）：

- 功法显示是否需要强制绑定法门名，还是允许“流派标签”先行占位。

## 14) 弟子信息展示模板（UI/名册对齐）

用于快速统一 UI 文案与系统判定口径，避免出现“等级/职业高低”表述。

- 概览行模板：`身份 · 修为 · 功法 · 技艺`
- 详情行模板：`身份: {身份} | 修为: {境界} | 功法: {主修法门} | 技艺: {主修技艺}`
- 缺省占位：未生成功法或技艺时，显示 `待定功法` / `待定技艺`（不回退为“职业/等级”）。
- 派遣判定说明：任务门槛由 `修为` 决定，路线倾向由 `功法` 决定，具体胜任度由 `技艺` 决定。

## 15) 弟子名册字段/数据结构草案（对齐 DisciplePanel / GameState）

用于后续 DisciplePanel 与存档字段对齐的最小结构草案，不代表最终存档格式。

最小字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `DiscipleId` | string | 唯一标识 |
| `Name` | string | 显示名 |
| `IdentityTag` | string | 身份标签（如 门人/执事/真传） |
| `RealmTag` | string | 修为境界标签（如 炼气/筑基/金丹/元婴/化神+） |
| `TechniqueTag` | string | 主修功法法门标签 |
| `SkillTag` | string | 主修技艺标签 |

可选扩展字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `AffiliationTag` | string | 归属峰堂/道统/外域标签（可空） |
| `SecondarySkillTags` | list<string> | 次要技艺标签（可空） |
| `RealmTierTag` | string | 三段式修为标签（初/中/高，便于概览） |
| `StatusTag` | string | 当前状态标签（如 空闲/执行/闭关/疗伤） |
| `OverviewText` | string | 概览文案缓存：`身份 · 修为 · 功法 · 技艺` |

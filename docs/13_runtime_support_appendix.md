# CountyIdle 运行支撑附录

> 本文用于保存地图、UI、存档、设计中扩展与停用系统等运行支撑信息。
> 若与 `docs/01_game_design_guide.md` 或 `docs/02_system_specs.md` 冲突，以主文档为准。

## 0. 文档职责

- 本文不裁决项目主轴。
- 本文不替代系统主规格。
- 本文只保存那些仍有价值、但不应进入主规格正文的运行支撑信息。

## 1. 地图与空间载体补充

以下内容可继续维护在本文：

- 天衍峰山门图的载体说明
- 世界图 / 外域图 / 二级地图的生成与入口补充
- 世界格类型、次级标签、区块权重、稀有度和伴生生成草案
- 地图与经营状态联动的表现细节

使用裁定：

- 地图是三层空间的观察与调度载体，不是独立玩法主轴。
- 地图补充细节若与主规格冲突，以 `docs/02_system_specs.md` 为准。

## 2. UI 与客户端设置补充

以下内容保留在本文：

- 主界面卷轴骨架与布局细节
- 弹窗家族、卷册式视觉语言、按钮态、提示条等表现层规则
- 客户端设置、分辨率、语言、快捷键、音量等支撑细节
- `治宗册 / 弟子谱 / 留影录 / 机宜卷 / 仓储账册` 等面板的具体布局说明

使用裁定：

- UI 的使命是组织判断，不是承载主逻辑。
- 表现层细节不得反向决定系统主法则。

## 3. 存档与运行兼容补充

以下内容保留在本文：

- `SQLite` 多槽结构
- 自动存档轮换
- 截图预览
- 复制槽位、排序筛选、卷册式存档面板等细节

使用裁定：

- 存档是运行支撑层，不是主设计主轴。
- 其详细结构可以在本文维护，但不应回占主规格正文。

## 4. 设计中系统补充

以下内容可继续单独推进：

- `DL-048` 世界格二级地图分层与入口系统
- `DL-050` 三相治宗循环系统
- 其他尚未进入当前运行版硬基线的扩展系统

使用裁定：

- 设计中内容必须明确标注“设计中 / 草案 / 未实现”。
- 不得以草案身份冒充当前运行版既成事实。
- 若设计中内容成熟，应先经 `01 -> 02 -> 实现` 的顺序回收。

## 5. 停用或历史系统补充

以下内容允许在本文保存历史上下文：

- 宗门弟子可视移动系统（停用）
- 历史地图联动细则
- 旧版 `Town / County / Prefecture` 技术命名与兼容说明

使用裁定：

- 这类内容用于迁移与恢复参考，不构成当前主线法则。
- 若恢复某停用系统，应重新经过 `01 -> 02 -> 实现` 的顺序，而不是直接照旧文复活。

## 6. 维护建议

- 主规格只保留当前版本必须成立的系统法则。
- 地图、UI、存档和设计中草案等支撑信息继续留在本文，不回灌主规格正文。
- 若本文再次膨胀，应继续按专题拆分，而不是让主规格重新失焦。

# CountyIdle 系统规格（初版）

> 用于统一“设计语言”，避免只写概念不落地。

## A. 郡城与人口系统

### 目标
- 提供稳定劳动力与增长曲线。

### 输入
- 粮食结余、幸福度、人口上限、外部威胁。

### 输出
- 人口变化、生产效率系数、流失/怠工事件。

### 关键规则（待填实数）
- 人口增长率 = `基础增长率 × 幸福度系数 × 粮食系数`
- 幸福度影响生产效率：`效率 = f(幸福度)`

### 边界条件
- 粮食不足时触发负增长或幸福下降。
- 幸福低于警戒线时触发流失。

## B. 职业与科技系统（士农工商）

### 目标
- 让玩家在生存、发展、科研之间做分配权衡。

### 输入
- 总人口、职业分配比例、资源库存。

### 输出
- 粮木石、金币、科研点、装备相关产能。

### 关键规则（待填）
- 农：基础资源产出公式
- 工：加工/锻造产出公式
- 商：资源转金币与幸福增益公式
- 士：科研点产出与科技解锁条件

## C. 精英繁育与血脉系统

### 目标
- 提供长期养成与“抽卡式惊喜”。

### 输入
- 父母属性、血缘关系、环境/建筑加成。

### 输出
- 后代八维属性、天赋词条、突变事件。

### 关键规则（待填）
- 子代属性 = `父母均值 + 随机波动`
- 近亲惩罚函数
- 突变概率与上限保护

## D. 装备与锻造系统

### 目标
- 构建中后期成长驱动力。

### 输入
- 图纸、材料、铁匠灵巧与天赋。

### 输出
- 品质、词条、耐久、熔炼产物。

### 关键规则（待填）
- 品质概率池（白绿蓝紫金）
- 词条池权重
- 耐久消耗与修复/熔炼回收比例

## E. 探险与守城系统

### 目标
- 用自动战斗输出稀缺资源，并对后方经营施压。

### 输入
- 英雄编队、装备、补给、当前层数、城防值。

### 输出
- 掉落、战损、层数推进、攻城事件结果。

### 关键规则（待填）
- 每层难度增长曲线
- Boss 机制与词条克制
- 攻城频率与强度计算

## F. 郡学突破（科研里程碑）系统

### 目标
- 将“士”职业产出的科研点转化为可见的中期增益，强化“职业分化 → 长线成长”反馈。

### 输入
- `Research`（科研点累计）
- 当前科技阶 `TechLevel`

### 输出
- 生产加成倍率（粮食/工产/贸易）
- 人口增长倍率
- 科技突破日志（用于玩家可观测反馈）

### 关键规则（v0.x）
- 每小时结算后根据 `Research` 判定可达科技阶：
  - T1（30）：粮食产量倍率 `1.15`
  - T2（90）：木石产量倍率 `1.15`，金币产量倍率 `1.10`
  - T3（180）：人口增长倍率 `1.20`
- 科技阶只升不降（同一存档内保持长期成长）。
- 产出公式示例：
  - `FoodGain = Farmers × BaseFood × FoodProductionMultiplier`
  - `GrowthRate = BaseGrowthRate × ... × PopulationGrowthMultiplier`

### 边界条件
- 默认倍率必须为 `1.0`，避免读档旧数据时出现 0 倍率。
- 科技倍率不可为负；若状态异常，结算时按最低 `1.0` 处理。

# 功能卡：底部控制台选中态靛青强调色（方案A）

## 功能信息

- 功能名：底部控制台选中态靛青强调色（方案A）
- 优先级：P2
- 目标版本：2026-03-10
- 关联系统：`BottomBar.tscn`、`PopupPanelBase.cs`、`Main*Panel.cs`

## 目标与用户收益

- 目标：提高底部快捷入口的选中态对比度与可识别性，明确当前打开的卷册/面板。
- 玩家可感知收益（10 分钟内）：一眼识别“当前处于哪个入口”，减少误触与回看成本。

## 实现范围

- 包含：底部快捷按钮选中态配色改为靛青；面板打开/关闭时按钮保持选中态；按钮悬停改为轻微放大并切换为靛青强调色；悬停触发轻微提示音。
- 不包含：布局改动、玩法逻辑调整、存档结构变更。

## 实现拆解

1. 底部控制台按钮 `pressed` 样式改为靛青并提升文字对比度。
2. 统一弹窗打开/关闭事件，联动底部按钮选中态。
3. 保持现有交互与绑定路径不变。

## 验收标准（可测试）

- [ ] 底部快捷按钮选中态为靛青底色，文字可清晰阅读。
- [ ] 打开/关闭“库房 / 中枢 / 谱系 / 弟子”面板时，按钮选中态同步更新。
- [ ] 玩法逻辑与存档结构无变更，运行不报错。

## 风险与回滚

- 风险：按钮切为 `toggle_mode` 后与现有交互产生误判。
- 回滚方式：恢复 `BottomBar.tscn` 旧样式与 `toggle_mode`，移除弹窗事件绑定。

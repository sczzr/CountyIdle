## 改动摘要

- 改动项：主界面六边形沙盘重构（十八轮补强）
- 改动前：主界面已整体卷轴化，但底部控制台、地图页签和左侧检视卡动作仍偏“统一描边按钮”，选中态不够明显，且检视卡残留 emoji 文案。
- 改动后：底栏与地图页签补齐 `hover / pressed` 卷册态，左侧检视卡动作文案统一回墨书语义，备用报表卡也回收到宣纸底色。

## 结果数据

- 指标 1：底栏/页签按钮状态=`单一描边 -> normal + hover + pressed`
- 指标 2：左侧检视卡动作文案=`emoji 提示 -> 墨线批令语义`
- 指标 3：备用报表卡底色=`深色卡 -> 宣纸卡`
- 指标 4：构建验证=`dotnet build .\\Finally.sln` 通过

## 结论

- 是否达到预期：`是`
- 下一步：`在 Godot 运行态确认页签选中态、底栏 toggle 态和 Tile Inspector 禁用态的视觉层次`

## 复盘

- 有效原因：这轮继续沿用“只改表现层，不动结算与交互主链”的方式，既把常驻控件的沉浸感补齐，也没有引入规则风险。
- 无效原因：`.tscn` 样式调整无法通过 `dotnet build` 覆盖运行态视觉细节，仍需要 Godot 烟测确认。
- 后续假设：若运行态验证通过，主界面常驻层就可视为已基本完成和弹窗家族一致的卷轴规范收口。

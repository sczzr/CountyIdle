## 基本信息

## 改动前

- 地图更多承担展示功能，经营压力变化不容易从地图直接读出来。
- 玩家无法在地图页直接发起灵道、坊市、附庸据点护持层面的调度。
- 地图与经营之间缺少“看得见 → 点得动 → 立刻反馈”的闭环。

## 改动后

- 新增地图状态条，按民心、威胁、住房、道路与资源态势显示 `繁荣 / 平稳 / 吃紧 / 紧张`。
- 世界/郡图/县城图接入状态色调变化：
  - 世界/郡图：背景、路线、节点、标题状态联动；
  - 县城图：地块/建筑色调与提示文案联动。
- 新增地图调度动作：
  - 郡图：整修驿路、赈济乡里；
  - 县城：修整街坊、夜巡清巷。
- 调度即时影响 `Happiness`、`Threat` 与资源库存。

## 指标结果

- 地图页切换后可直接观察当前经营态势变化。
- 地图调度成功/失败均有日志反馈，且即时刷新到地图和顶部资源面板。
- 调度与地图表现均未破坏小时结算主循环。

## 结果结论

- 本轮达成 `DL-013` 一期目标：经营状态已映射到地图表现，地图页也能反向调度经营指标。
- 后续可以继续扩展为随机地图事件链、调度冷却与持续型治理 Buff。

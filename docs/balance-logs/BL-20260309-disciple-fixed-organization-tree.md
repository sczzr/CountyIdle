# 数值平衡日志：弟子谱固定宗门组织树

- 任务 ID：`DL-043`
- 时间：`2026-03-09`

## 改动前

- 左侧名册已经支持树状展开，但峰脉与堂口仍较依赖弟子关联文本即时解析；
- 峰内层级存在，但组织语义还不够稳定，后续难以直接承接峰内管理。

## 改动后

- 左侧名册优先按固定宗门组织树归类，不再主要依赖关联文本首段；
- 树层级细化为：`峰脉 -> 堂口 / 机构 -> 条线 / 班序 -> 册级 -> 弟子`；
- 不同职司被稳定收口到更具体的峰内条线，如 `总枢亲传线 / 阵枢营造线 / 护山检修线 / 推演研修线 / 商路采办线`；
- 右侧档案刷新、筛选、排序与地图联动能力保持不变。

## 指标结果

- 名册树层级：`5`
- 组织归类方式：`固定宗门树优先`
- 新增层级：`条线 / 班序`
- 构建结果：`dotnet build .\Finally.sln` 通过

## 结论

这轮仍属于表现层与组织管理结构增强，不改人口、资源、战斗与治理数值。  
它让弟子谱更接近“可管理的宗门组织架构”，后续若继续接任命、峰务指派、师承关系或考核记录，会更容易顺着这棵固定组织树往下落。

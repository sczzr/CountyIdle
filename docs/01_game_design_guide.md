# CountyIdle 游戏开发总指南

## 1. 愿景（不轻易变）

- 类型：`SIM + Idle + RPG`
- 视角：一郡之守，宏观治理 + 微观精英养成
- 主题：文明演进、血脉传承、资源循环、庇护与征服
- 核心体验：挂机有成长，决策有反馈，长期有目标

## 2. 核心飞轮（所有设计都要服务）

1. 人口繁衍
2. 职业分化（士农工商）
3. 精英繁育
4. 武装探险
5. 反哺郡县并进入下一轮

任何新功能都必须回答：
- 它强化了飞轮的哪一环？
- 它会不会打断飞轮节奏？
- 它的收益是否可被玩家感知？

## 3. 设计边界

- `Main.cs` 只承担 UI 与交互组织。
- 玩法计算放在 `scripts/systems`，由 `scripts/core/GameLoop.cs` 调度。
- 数值改动优先小步迭代，避免一次性重写多系统。
- 新增机制必须能存档、可观测（日志/状态变化可见）。

## 4. 里程碑拆分建议

### M1：可玩闭环（当前阶段）
- 人口、资源、职业、自动探险、存档读档可用
- 目标：玩家能连续挂机 30 分钟且系统稳定

### M2：成长深度
- 科技树第一版、装备品质与词条、精英繁育增强
- 目标：形成“培养方向差异”

### M3：长期目标
- 攻城战、Boss 克制、后期地图与稀有掉落
- 目标：形成中后期策略分流

## 5. 开发优先级原则

- 优先修复破坏飞轮的问题（卡死、崩溃、资源负循环）。
- 再做“可感知收益高”的优化（UI反馈、日志、节奏感）。
- 最后做低影响装饰项（表现层微调）。

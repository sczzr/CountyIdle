# CountyIdle 游戏设计总纲

## 1. 游戏定位（不轻易变）

- 类型：`SIM + Idle + RPG`
- 背景：以 **浮云宗（青云州江陵府）** 为原型的天衍峰经营、外域经营与历练成长
- 玩家身份：掌门 / 宗主，当前直接经营焦点为 **天衍峰**，负责统筹山门、门人、资源、阵务、传承与外务
- 核心体验：
  - 挂机时，宗门会持续产出门人、灵材、研究进度与江陵府外域变化
- 决策时，玩家通过宗主中枢、内务建设、双轨货币结算、传承路线与历练安排塑造发展方向
  - 长期目标指向“宗门崛起、护山立威、真传成长、问道飞升”
- 世界观基线：对外文档与文案统一使用“浮云宗·天衍峰”语义；若出现 `County / Town / Prefecture`，仅视为代码层历史技术名

## 2. 天衍峰经营背景的核心叙事

这不是“经营一座县城”的游戏，而是“经营 **浮云宗天衍峰** 这一支柱山峰，并通过青云峰三大总殿协同全宗”的修仙宗门生态。

玩家主要在三层空间做决策：

1. **天衍峰内部**：阵材圃、傀儡工坊、传法院、庶务殿、弟子作息、任务法旨与职司摘要
2. **青云峰与外域协同**：内务总殿、外事总殿、传功总殿、附庸据点、府域灵道、商路与威胁
3. **长期成长**：阵法传承、灵根苗子、真传培养、法器傀儡、历练与护山战备

## 3. 核心飞轮（所有设计都要服务）

1. 门人生息与附庸汇聚
2. 产业供养与灵材采炼
3. 传承研修与技艺推演
4. 职司分流与转任成长
5. 灵根苗子与优秀后辈培育
6. 真传 / 英雄单位历练
7. 装备、法器与战备成长
8. 弟子与玩家共同建设宗门
9. 宗门护持外域附庸据点、山道与灵脉，换取稳定、资源与声望
10. 宗门发展继续反哺门人、产业、研究与历练，进入下一轮

任何新功能都必须回答：

- 它强化了飞轮的哪一环？
- 它是否让“天衍峰经营感”更强，而不是更像抽象数值堆叠？
- 它是否破坏了挂机积累与主动决策之间的节奏平衡？
- 玩家是否能在 10 分钟内感知到这次改动的收益？

## 4. 关键管理对象（设计时必须想清楚）

### 4.1 门人不是纯数字

当前代码层仍有 `Population` 这样的技术字段，但设计语义上，它代表的是宗门人口池，包含：

- 凡俗依附者 / 杂役
- 记名与外门弟子
- 内门弟子与执事
- 真传、长老、客卿等高阶战力预留层

短期内可以抽象结算，长期要逐步提升可解释性与层次感。

### 4.1.1 玩家发放任务，而不是逐人点岗

- 玩家作为宗主，主要决策单位应是“法旨 / 任务单”，而不是直接拨动四类岗位人数。
- 任务法旨需要让玩家看懂“我在让天衍峰做什么”，例如阵材采炼、阵枢营造、巡山警戒、阵法推演、总坊值守、外事行商。
- 系统可以继续保留四类职司字段作为兼容层，但玩家可见交互应收口到任务面板与职司摘要。
- 宗门内部建设、锻器与驻地调度要通过 `贡献点 + 灵石` 双轨结算；宗门外交易只使用灵石。

### 4.2 产业不是普通城市工业

当前“产业系统”对外应始终解释为：

- 阵材圃、傀儡工坊、传法院、青云总坊、庶务殿等天衍峰 / 青云峰协同设施
- 目标是供养门人、支撑护山大阵、傀儡战备、峰内研修与外域协同，而非单纯追求工业化产值

### 4.3 外域不是行政区，而是宗门势力范围

外域地图与附庸圈层的意义应是：

- 资源来源
- 历练路径
- 外患压力
- 浮云宗影响力延伸
- 护持与反哺关系

## 5. 设计边界

- `Main.cs` 只承担 UI 与交互组织，不承载复杂玩法计算
- 玩法计算放在 `scripts/systems`
- 主循环调度由 `scripts/core/GameLoop.cs` 统一串行处理
- 新增机制必须满足：
  - 可存档
  - 可观测
  - 可解释
  - 不破坏 `60` 分钟一小时的结算节奏
- 新设计优先补“天衍峰经营感”和“世界观一致性”，不做无根由的城市经营残留文案

## 6. 核心价值观（必须遵守）

- 不允许职业歧视叙事或机制：职司体现分工，不体现人的高低贵贱
- 不把门人、附庸、外域附庸据点当作可随意消耗的“无人格资源”
- 游戏冲突应服务于“守护、发展、秩序与选择”，而非无意义伤害
- 宗门系统的最终目标是“共同建设、共同受益、共同护持”
- 文案与机制均须遵守人类基本准则与道德规范

## 7. 里程碑拆分建议

### M1：宗门经营可玩闭环（当前阶段）

- 门人、资源、职司、基础研究、自动历练、存档读档可用
- 宗主中枢、贡献点、职司摘要可用
- 目标：玩家能以“管理宗门”的视角连续挂机 `30` 分钟且系统稳定

### M2：传承与战备深度

- 传承树第一版、装备/法器体系、灵根苗子培养增强、真传成长
- 目标：形成“不同宗门路线”而非单一路数值堆高

### M3：外域与护山长期目标

- 护山战、Boss / 妖王克制、后期外域与洞天稀有掉落
- 目标：形成中后期修仙策略分流与宗门身份差异

## 8. 开发优先级原则

- 优先修复破坏飞轮的问题：卡死、崩溃、资源负循环、存档破坏
- 再做高可感知收益优化：宗门术语统一、日志解释、视觉反馈、节奏感
- 最后做低影响装饰项：纯表现微调、资产替换、非关键动画

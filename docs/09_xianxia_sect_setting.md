# CountyIdle 宗门设定与术语基线

> 本文用于统一“当前项目到底在讲一个什么样的修仙宗门故事，以及旧技术词在新世界观里该怎么解释”。
>
> 若 `09` 与旧归档正文存在表述差异：**世界观与玩家可见语义以本文为准**。  
> 若 `09` 与系统公式冲突：**规则与数值边界以 `docs/02_system_specs.md` 为准**。

## 1. 一句话设定

玩家扮演 **浮云宗** 的掌门 / 宗主。宗门坐落于 **苍玄界·青云州·江陵府**，当前可玩经营视角聚焦在 **天衍峰**，并与青云峰上的 **内务总殿 / 外事总殿 / 传功总殿** 协同运转：在山门中管理门人、阵务、传承与库藏，在外域中经营附庸关系、府域灵道、资源与威胁，并通过历练、培养与护山逐步壮大宗门。

## 1.1 默认宗门原型（强制）

- 宗门名：`浮云宗`
- 所在地：`青云州 · 江陵府`
- 权力中枢：`青云峰`
- 三大总殿：
  - `内务总殿`：全宗档案、贡献点总账、律法与资源调拨中枢
  - `外事总殿`：附庸、商路、外交与青云总坊协调中枢
  - `传功总殿`：入门功法、核心传承与高阶讲道中枢
- 当前经营主视角：`天衍峰`
- 天衍峰核心定位：`战略防御与指挥中心`
- 天衍峰核心堂口：`阵堂`

## 1.2 当前可玩视角解释

- 玩家仍保留“宗主 / 掌门”级调度权限；
- 但主界面上的设施、任务法旨、职司摘要与地图场所，默认按“浮云宗天衍峰日常经营面”理解；
- 因此系统里的庶务、研究、营造、交易等条目，应优先映射到 **天衍峰阵堂、庶务殿、传法院、青云总坊接口**，而不是抽象泛宗门空壳设定。

## 2. 玩家到底在管理什么

### 2.1 山门内部

- 住房与基础供给
- 宗主中枢与执事调度
- 阵材圃、傀儡工坊、传法院、青云总坊接口、庶务殿等天衍峰核心设施
- 门人作息、通勤、任务执行、修养与恢复
- 库藏、材料链、装备与法器
- 灵石与贡献点的双轨内务流通

### 2.2 外域周边

- 外域附庸据点与道路
- 资源来源与商路
- 袭扰、妖患与威胁压力
- 宗门影响力与护持关系

### 2.3 长期成长

- 传承研修与技艺推演
- 灵根苗子与优秀后辈培养
- 真传 / 英雄单位历练
- 护山战备与后期首领挑战

### 2.4 组织结构原型（当前默认）

| 层级 | 默认机构 | 当前对外理解 |
| --- | --- | --- |
| 宗门中央 | 青云峰 | 浮云宗最高权力与总账中枢 |
| 中央行政 | 内务总殿 / 外事总殿 / 传功总殿 | 分别负责总账内务、对外协调、传承审核 |
| 峰级经营 | 天衍峰 | 当前玩家主要经营与观察的山门焦点 |
| 峰级核心堂口 | 阵堂 | 护山大阵、傀儡、战争推演与阵法研发 |
| 峰级日常机构 | 庶务殿 / 传法院 / 青云总坊接口 | 对应内务调拨、阵法研修、内外交易协同 |

## 3. 宗门人口池的叙事解释

当前系统里仍有 `Population` 这类技术字段。对外语义上，它不再等于“城市人口”，而是**宗门人口池**，是以下群体的抽象总和：

- 凡俗依附者与杂役
- 记名弟子与外门弟子
- 内门弟子与执事
- 真传、长老、客卿等更高层战力预留

说明：

- 现阶段仍以聚合数值结算为主；
- 未来若做更细分的弟子层级、转任、真传系统，应在不破坏存档的前提下逐步显式化；
- 任何新文档都不应再把 `Population` 直接写成“县民 / 城民 / 百姓总量”。

## 4. 当前统一术语表（强制）

| 技术名 / 旧说法 | 当前对外说法 | 说明 |
| --- | --- | --- |
| `Population` / 人口 | 门人总量 / 宗门人口池 | 泛指浮云宗生态中的可调配人口总量 |
| `Farmers` | 阵材职司汇总 | 由 `阵材采炼` 等天衍峰任务自动折算得到 |
| `Workers` | 阵务职司汇总 | 由 `阵枢营造 / 巡山警戒` 等任务自动折算得到 |
| `Merchants` | 总坊 / 外事职司汇总 | 由 `总坊值守 / 外事行商` 等任务自动折算得到 |
| `Scholars` | 推演职司汇总 | 由 `阵法推演` 等任务自动折算得到 |
| `ContributionPoints` / 贡献点 | 宗门贡献点 | 只在宗门内部流通，用于内务建设、调度与奖励 |
| `Research` / 科技 | 传承研修 / 功法技艺推演 | 当前仍可保留“科技树”技术叫法，但对外更偏修仙传承 |
| `Breeding` / 精英繁育 | 灵根苗子培养 / 后辈培育 | 可包含血脉、资质、突变、优秀后辈等解释 |
| `Combat` / 探险 | 外务历练 / 护山战备 | 既包括外域历练，也包括宗门武备准备 |
| `Equipment` | 装备 / 法器 | 当前实现仍以装备系统为主，对外可逐步修仙化 |
| `CountyEvent` | 宗门见闻 / 山门事件 | 坊市、讲法、袭扰等动态事件 |
| `CountyTown` | 浮云宗驻地 / 天衍峰山门图 | 历史技术名，不是县城设定 |
| `Prefecture` | 江陵府外域 / 附庸圈层 | 历史技术名，不是行政郡府设定 |
| `World` / 世界图 | 世界地图 | 用于观察更高层的地貌、灵脉、道路与势力态势 |

## 5. 文案门禁（新文档必须遵守）

### 5.1 默认要用的说法

- 用“宗门 / 山门 / 驻地”，不用“县城 / 县治”
- 用“浮云宗 / 青云峰 / 天衍峰 / 阵堂 / 庶务殿 / 传法院 / 青云总坊”这些有组织结构来源的名称，不用空泛占位名
- 用“外域 / 附庸据点 / 灵道”，不用“郡图 / 州府 / 郡县治理”
- 用“门人 / 弟子 / 附庸”，不用“城民 / 百姓”
- 用“宗主中枢 / 治理方略 / 执事调度”，不用“直接调岗位人数”作为主要表述
- 用“职司 / 转任”，不用“职业 / 转职”作为唯一说法
- 用“传承 / 研修 / 推演”，不用纯行政或工业化语境
- 用“贡献点 + 灵石双轨内务”，不用“一切都只花金钱”的单轨表述

### 5.2 允许保留旧词的场景

仅在以下场景允许保留旧技术命名：

- 引用具体代码类名、文件名、字段名
- 解释历史归档与旧 topic
- 说明兼容层或迁移关系

此时必须使用反引号包裹，例如：`CountyEventSystem`、`CountyTownMapViewSystem`。

## 6. 设计边界（避免假修仙、真县城）

- 天衍峰经营不等于把县城文案直接替换成仙侠名词
- 新功能需要同时回答“管理对象是谁”“为什么是宗门会关心这件事”
- 外域设计强调护持、互惠、资源与威胁，不走纯行政治理叙事
- 冲突系统强调守护、抉择与代价，不鼓励去人性化表述

## 7. 文档更新规则

当新增系统或做世界观升级时，建议按以下顺序更新：

1. 先改 `docs/09_xianxia_sect_setting.md`
2. 再改 `docs/01_game_design_guide.md`
3. 再改 `docs/02_system_specs.md`
4. 最后回写 `docs/05_feature_inventory.md` 与 `docs/08_development_list.md`

这样可以避免“规则已经是宗门系统，但文档还像县城经营”的语义撕裂。

# 改动提案（Change Proposal）

## 提案信息

- 标题：顶栏时间改为架空农历 + 二十四节气，并扩展三档倍率
- 日期：2026-03-07
- 提案人：Codex
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：顶部仍使用“纪元 / 周期 / 结算倒计时”表达，题材感不足，且倍率只有 `x1/x2`，节奏调节空间偏小。
- 证据（数据/玩家反馈）：当前界面直接显示“纪元 Ⅰ · 第 1 周期”“距离大暑结算还有 59 秒”，与用户想要的“某某年、某月、某日 + 节气”表达不一致。

## 改动内容

- 改什么：
  - 新增架空农历换算：`12 月 * 30 日 * 24 时`
  - 每 `15` 日对应一个节气，全年接入 `24` 节气
  - 顶栏改为显示“景禾某年 某月 某日”，Legacy 布局额外显示节气与时辰
  - 移除结算倒计时文案，保留节气进度条
  - 倍率扩展为 `x1/x2/x4`
  - 读档后恢复小时内结算相位，不从整点重置
- 不改什么：
  - 不改 `1 秒现实时间 = 1 游戏分钟`
  - 不改 `60 游戏分钟` 小时结算顺序
  - 不改 `GameState` 存档字段结构
- 影响系统：
  - `scripts/systems/GameCalendarSystem.cs`
  - `scripts/Main.cs`
  - `scripts/core/GameLoop.cs`
  - `scenes/ui/TopBar.tscn`
  - `scenes/ui/BottomBar.tscn`
  - `scenes/ui/figma/*.tscn`

## 预期结果

- 预期提升指标：
  - 时间展示更符合古风经营题材
  - 玩家能直接理解日期与节气，而不是抽象周期编号
  - 节奏控制从 2 档扩展为 3 档
- 可接受副作用：
  - 架空农历不是现实农历精确换算，而是“农历风格 + 节气融入”的固定历法

## 验证计划

- 验证方式：
  - 构建项目并检查 UI 节点绑定
  - 手动检查 `x1/x2/x4` 按钮与快捷键轮换
  - 校验读档后分钟相位延续与小时结算节奏不变
- 观察周期：单次 5~10 分钟交互烟测 + 至少一次读档验证
- 成功判定阈值：
  - 不再显示结算倒计时
  - 时间显示稳定更新为架空农历日期与节气
  - `x4` 不导致主循环报错或居民动画异常

## 回滚条件

- 触发条件：出现时间显示错位、读档后结算相位错乱、或 `x4` 导致明显运行异常。
- 回滚步骤：
  1. 回退 `GameCalendarSystem` 与 UI 顶栏/底栏改动
  2. 将 `GameLoop` 最高倍率恢复为 `x2`
  3. 恢复旧版时间显示文案

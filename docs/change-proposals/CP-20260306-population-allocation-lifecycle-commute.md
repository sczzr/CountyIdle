# 改动提案（Change Proposal）

## 提案信息

- 标题：人口分配接入生活约束与步行通勤机制
- 日期：2026-03-06
- 提案人：System Agent
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：人口仅按“食物/住房/幸福”做总量增长，缺少生老病死、睡眠、衣物与通勤对劳动供给的影响。
- 证据（数据/玩家反馈）：现有系统中“住房与岗位距离”“睡觉后再上工”等体验无法表达，导致人口与岗位关系偏抽象。

## 改动内容

- 改什么：
  - 新增人口分层（幼年/成年/老年）与病患状态
  - 新增住房睡眠承载与衣物覆盖规则
  - 新增出生/衰老/患病/死亡小时聚合结算
  - 新增步行通勤时长与到岗率因子
  - 将情绪修正扩展为“睡眠+衣物+通勤+疾病”
- 不改什么：
  - 不改变 `1 秒=1 分钟` 与 `60 分钟`小时结算节奏
  - 不实现个体级实时寻路
  - 不一次性上线纺织产业整条产线
- 影响系统：
  - `docs/02_system_specs.md`（人口分配规则）
  - `scripts/models/GameState.cs`
  - `scripts/systems/PopulationSystem.cs`
  - `scripts/systems/EconomySystem.cs`
  - `scripts/systems/PrefectureMapGeneratorSystem.cs`（距离输入复用）
  - `scripts/Main.cs`（日志与基础展示）

## 预期结果

- 预期提升指标：
  - 人口变化可解释性提升（日志可分解为出生/衰老/死亡/病患恢复）
  - 住房与通勤的建设价值提升（可直接影响有效岗位）
  - 人口系统与产业系统耦合增强，飞轮反馈更真实
- 可接受副作用：
  - 早期增长速度可能小幅下降，需要后续平衡参数微调

## 验证计划

- 验证方式：
  - `dotnet build .\\Finally.sln`
  - 运行 2~3 小时结算窗口，观察到岗率、疾病与人口变动日志
  - 人为拉高通勤距离/压低住房，确认产出与幸福按预期下滑
  - 存档/读档后新增字段保持兼容
- 观察周期：10~20 分钟实机
- 成功判定阈值：不出现负值/崩溃，且通勤与住房变化可稳定映射到有效岗位变化

## 回滚条件

- 触发条件：连续 `6` 小时出现“有效岗位显著下降 + 食物/金币持续恶化”。
- 回滚步骤：
  1. 保留新增字段与日志结构
  2. 停用 `onDutyFactor * sleepFactor * healthLaborFactor` 对岗位的乘算
  3. 回退人口系统到当前增长与幸福规则

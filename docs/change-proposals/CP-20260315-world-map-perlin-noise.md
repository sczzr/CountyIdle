# 改动提案（Change Proposal）

## 提案信息

- 标题：世界地图/地形基础噪声切换为柏林噪声
- 日期：2026-03-15
- 提案人：Codex
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：世界地图的基础噪声使用的是 value noise 分形叠加，地形过渡与宏观结构可进一步更自然、连贯。
- 证据（数据/玩家反馈）：需求提出“世界地图使用柏林噪声来生成地图和地形”。

## 改动内容

- 改什么：
  - 将 `XianxiaWorldGeneratorSystem` 的基础噪声采样改为柏林噪声（Perlin noise）实现。
  - 维持现有噪声频率、叠加层数与持久度参数不变，仅替换基础噪声类型。
- 不改什么：
  - 山脉、河流、龙脉、遗迹、资源与道路的生成流程不改。
  - 地图尺寸、种子与配置字段不改。
- 影响系统：
  - `CountyIdle/scripts/systems/XianxiaWorldGeneratorSystem.cs` 的基础噪声采样与 `GenerateBaseFields` 相关分布。

## 预期结果

- 预期提升指标：
  - 地形高度与气候分布更平滑、连贯。
  - 生物群系边界更自然，减少突兀拼接感。
- 可接受副作用：
  - 相同种子下地形分布将发生变化。
  - 个别生物群系占比可能出现小幅偏移，需要调参观察。

## 验证计划

- 验证方式：
  - 运行世界地图生成，目视检查宏观连贯性。
  - 记录基本分布（高海拔/低海拔占比、主要生物群系占比）与旧版对比。
- 观察周期：本地一次完整生成与 UI 预览确认。
- 成功判定阈值：
  - 生成无异常、无明显断裂带或过度碎片化区域。
  - 生物群系占比在可接受区间内（±10% 以内浮动为默认可接受）。

## 回滚条件

- 触发条件：
  - 出现大面积不可通行地形、生态分布异常、或生成性能显著下降。
- 回滚步骤：
  - 恢复原 `SampleValueNoise` 采样路径或回退此改动提交。

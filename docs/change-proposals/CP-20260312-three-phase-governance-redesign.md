# 改动提案（Change Proposal）

## 提案信息

- 标题：三相治宗循环重设计（季度战略相位 + 双层时间制）
- 日期：2026-03-12
- 提案人：System Agent
- 变更级别：`L3 架构`

## 改动背景

- 当前问题：多数系统虽然都在运转，但玩家中期决策常表现为“加成项堆叠”，路线辨识度不足；同时当前单层分钟制不利于表现修仙题材应有的长期岁月感。
- 证据（数据/玩家反馈）：现行治理项更偏小时结算加减，季度层面的战略切换缺少明确机会成本与风险反馈，导致“选哪条都差不多”；以 `1 秒 = 1 分钟` 来承载闭关、成长、扩张等修仙长期行为时，时间感偏慢且不够自然。

## 改动内容

- 改什么：
  - 在文档层引入 `季度立纲 -> 月度筹划 -> 小时结算` 三相节奏；
  - 为整体设计引入“双层时间制”，区分 `细时间运转` 与 `长时间推进`；
  - 定义战略相位（保守蓄势 / 务实扩张 / 外务远征）对人口、资源、研修、威胁、历练收益的联动权重；
  - 补齐相位切换冷却、失衡惩罚、日志观测点、正式时间口径与回滚阈值；
  - 产出最小实现闭环工单，后续按单相位先行接入。
- 不改什么：
  - 不在本提案直接改动全量系统代码；
  - 不新增破坏旧档兼容的存档字段；
  - 不同步推进与该主目标无关的 UI 美术大改。
- 影响系统：
  - `docs/01_game_design_guide.md`
  - `docs/02_system_specs.md`
  - `docs/05_feature_inventory.md`
  - `docs/08_development_list.md`
  - 后续实现预计影响：`GameState`、`SectGovernanceSystem`、`GameLoop`、`Main`、治理相关 UI 面板

## 预期结果

- 预期提升指标：
  - 玩家在 `10` 分钟内可观测到战略相位差异（至少 `2` 个核心指标方向发生偏移）；
  - 玩家能理解“眼前正在发生的事”和“宗门几月几季的成长”是两层不同节奏；
  - 季度末可从日志读到“收益 - 代价”闭环，而不是单向增长；
  - 中期决策从“调参数”升级为“选路线 + 承担机会成本”。
- 可接受副作用：
  - 初期学习成本略升；
  - 短期需要增加引导文案与日志解释密度。

## 验证计划

- 验证方式：
  - 文档一致性检查：`01/02/05/08` 术语、飞轮、目标一致；
  - 时间一致性检查：明确“细时间 / 长时间 / 日月季推进”的正式口径没有混写；
  - 实现后运行验证：`dotnet build .\Finally.sln` + Godot `F5`；
  - 观察 `6` 次小时结算 + `1` 次月度结算，确认相位差异可观测且无负值异常。
- 观察周期：一次完整季度（`90` 游戏日）中的至少前 `30` 日。
- 成功判定阈值：相位切换后 `Population / Contribution / Threat / Research / Expedition收益` 至少出现 `3` 项可解释变化，且不破坏存档兼容。
  - 双层时间制相关说明能让玩家读懂“细时间 / 长时间 / 当前实现状态”的区别，不造成文档歧义。

## 回滚条件

- 触发条件：
  - 连续 `6` 次小时结算出现负循环（资源持续耗尽或威胁失控）；
  - 相位切换后差异不可观测（指标波动低于当前版本噪声）。
- 回滚步骤：
  1. 将运行时策略回退到现有治理加成路径；
  2. 将时间规则整体回退到上一版稳定口径；
  3. 保留本提案文档并记录失败原因；
  4. 缩小变更面，按“单相位试点 + 单系统接入”重新推进。

# 改动提案（Change Proposal）

## 提案信息

- 标题：L2 道路与河流改为独立绘制装饰层
- 日期：2026-03-12
- 提案人：Codex
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：现有文档将 `Layer 2` 的道路 / 河流默认定义为 `hex` 六方向连接资产，容易把视觉设计收束成“每个格心向六边扩散”的拼片逻辑，和当前想要的自由卷轴笔触表达冲突。
- 证据（数据/玩家反馈）：最新需求已明确“道路和河流不与六边形地图产生关联，只需单独绘制一条道路和一条河流，不锚定在六边形框架中”。

## 改动内容

- 改什么：
  - 将 `Layer 2` 中的道路 / 河流改为独立绘制的装饰性资源；
  - 资源默认以单条道路、单条河流为出图单位，不再要求 `4x2 hex connector sheet`；
  - 运行时文档改为支持自由摆放、拉伸、旋转或样条跟随的装饰层思路。
- 不改什么：
  - 不调整 `Layer 1` 基础地块仍为 hex 底盘的既有定位；
  - 不移除历史 connector 方案的兼容记录；
  - 不在本轮直接修改运行时代码。
- 影响系统：
  - `DL-049` 地图素材分层资产流水线
  - `docs/11_map_asset_production_spec.md`
  - `docs/14_map_layer_rendering_implementation_plan.md`
  - `docs/15_map_layer2_nanobanana_prompt_spec.md`

## 预期结果

- 预期提升指标：
  - L2 出图约束明显降低，美术可更快产出首版道路 / 河流资源；
  - 地图画面更接近卷轴国风笔触，而非标准 hex 拼片感；
  - 后续世界 / 二级地图若采用不同形态道路，也不必重新出整套 connector sheet。
- 可接受副作用：
  - 运行时自动拼接能力会弱于严格 bitmask 方案；
  - 若需要高度程序化路径系统，后续可能要补独立摆放规则或控制点规范。

## 验证计划

- 验证方式：
  - 检查 `11 / 14 / 15 / 08 / 05` 文档是否统一为“独立道路 / 独立河流”口径；
  - 检查是否仍存在把道路 / 河流定义为默认 hex connector 的描述。
- 观察周期：本轮文档更新后立即生效，后续首次 L2 美术出图时复核。
- 成功判定阈值：相关文档主口径一致，且美术无需再按 hex 框架准备道路 / 河流资源。

## 回滚条件

- 触发条件：
  - 后续运行时验证证明自由摆放方案无法满足当前地图可读性或接入效率；
  - 世界图 / 二级地图必须恢复强逻辑拼接，且独立资源方案造成明显返工。
- 回滚步骤：
  - 将 `Layer 2` 默认方案恢复为 connector sheet；
  - 保留本轮自由摆放方案作为宗门图专用特例或概念资产方案。

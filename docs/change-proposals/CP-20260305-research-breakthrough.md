# 改动提案（Change Proposal）

## 提案信息

- 标题：科研里程碑驱动的郡学突破
- 日期：2026-03-05
- 提案人：Gameplay Agent
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：科研点仅累计展示，短中期缺少可感知反馈，导致“士”职业吸引力不足。
- 证据（数据/玩家反馈）：现有结算中 `Research` 仅增加数值，不影响任何产出或增长公式。

## 改动内容

- 改什么：
  - 新增 3 档科研阈值，解锁后提供永久倍率
  - 科技阶与倍率写入 `GameState`，确保可存档
  - 通过日志提示突破结果
- 不改什么：
  - 不新增科技树界面与分支选择
  - 不改探险战斗数值
- 影响系统：
  - `models/GameState`
  - `systems/EconomySystem`
  - `systems/PopulationSystem`
  - `systems/ResearchSystem`（新增）
  - `core/GameLoop`
  - `Main` 摘要展示

## 预期结果

- 预期提升指标：
  - 10 分钟内出现首个突破日志
  - 粮食增长速度在 T1 后可见提升（同岗位下每小时 +15%）
  - 人口增长在 T3 后加速（同状态下增长人数更稳定）
- 可接受副作用：
  - 中前期资源增长略快，可能降低早期饥荒压力

## 验证计划

- 验证方式：
  - `dotnet build .\Finally.sln`
  - 运行观察 10+ 分钟结算日志与资源曲线
  - 存档后读档验证科技阶与倍率保持
- 观察周期：单次 10~30 分钟挂机窗口
- 成功判定阈值：至少触发 T1，且读档后科技阶不回退

## 回滚条件

- 触发条件：资源增长导致核心飞轮失衡（岗位决策价值显著下降）
- 回滚步骤：
  1. 停用 `ResearchSystem` 调度
  2. 恢复所有倍率为 `1.0`
  3. 记录回滚原因并进入下一轮提案

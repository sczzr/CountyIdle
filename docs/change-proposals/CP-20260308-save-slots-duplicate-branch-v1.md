# 改动提案（Change Proposal）

## 提案信息

- 标题：多存档槽位复制分支（V1）
- 日期：2026-03-08
- 提案人：Gameplay Agent
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：
  - 多槽系统已有手动槽、自动槽、筛选排序和截图预览，但玩家要“从某个历史点分支”时仍要先读档再另存；
  - 这会增加额外操作，也更容易误把当前进度覆盖到错误槽位。
- 证据（数据/玩家反馈）：
  - `docs/05_feature_inventory.md` 中多存档界面下一步仍是“更完整的管理入口”。

## 改动内容

- 改什么：
  - 新增“复制所选槽”管理入口
  - 从所选槽最新快照直接复制出新手动槽
  - 若存在截图预览，则同步复制对应 PNG
- 不改什么：
  - 不改变 SQLite 表结构
  - 不改变默认槽/自动槽保护规则
  - 不实现外部文件导入导出
- 影响系统：
  - `CountyIdle/scripts/core/SaveSystem.cs`
  - `CountyIdle/scripts/ui/SaveSlotsPanel.cs`
  - `CountyIdle/scripts/ui/MainSaveSlotsPanel.cs`
  - `CountyIdle/scenes/ui/SaveSlotsPanel.tscn`

## 预期结果

- 预期提升指标：
  - 玩家创建实验分支存档的路径更短
  - 自动档历史节点更容易转为长期保留手动档
  - 多槽系统更接近完整管理闭环
- 可接受副作用：
  - 列表中可能出现更多相近名称槽位

## 验证计划

- 验证方式：
  - 选择主存档、自动存档、手动槽分别执行一次复制
  - 验证新槽位可读取、重命名、删除
  - 验证预览图有值时会同步复制
  - `dotnet build .\Finally.sln`
- 观察周期：一轮启动 + 数次面板内复制操作
- 成功判定阈值：
  - 复制不影响原槽位数据
  - 新槽位可完整参与现有多槽流程

## 回滚条件

- 触发条件：
  - 复制后新槽位无法读取
  - 复制入口导致保护槽规则失效
- 回滚步骤：
  1. 回退复制按钮与事件绑定
  2. 回退 `SaveSystem` 槽位复制逻辑
  3. 保留当前多槽、预览、筛选排序功能

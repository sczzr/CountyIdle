# 改动提案（Change Proposal）

## 提案信息

- 标题：仓储链路专用管理界面接入
- 日期：2026-03-06
- 提案人：Gameplay Agent
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：仓储和矿产链路数据分散在顶部资源栏、岗位状态与报表卡片中，缺乏集中管理入口。
- 证据（数据/玩家反馈）：新增需求明确要求“需要有专门的仓库管理界面”。

## 改动内容

- 改什么：
  - 新增 `仓库管理` 独立页签
  - 新增仓储总览、库存状态、加工链状态面板
  - 新增面板快捷操作：矿仓联建、制工具、扩建工坊、扩建官署
  - `Main.cs` 接入页签切换与实时状态刷新
- 不改什么：
  - 不新增 `GameState` 字段
  - 不改核心结算顺序与公式
  - 不改 Figma 布局结构
- 影响系统：
  - `CountyIdle/scenes/ui/WorldPanel.tscn`
  - `CountyIdle/scripts/Main.cs`
  - `docs/02_system_specs.md`

## 预期结果

- 预期提升指标：
  - 仓储管理信息集中可见，降低操作切页成本
  - 矿仓联建与制工具操作可在同一页面完成
  - 玩家可更快识别仓储压力和材料瓶颈
- 可接受副作用：
  - 顶部页签数量增加，低分辨率下横向空间更紧张

## 验证计划

- 验证方式：
  - `dotnet build .\Finally.sln`
  - 运行检查仓库页签切换与状态刷新
  - 验证仓库页按钮触发日志与资源变化
- 观察周期：10 分钟交互烟测
- 成功判定阈值：无空引用、无切页异常、无主循环中断

## 回滚条件

- 触发条件：仓库页签导致主界面交互明显退化或出现高频异常
- 回滚步骤：
  1. 移除仓库页签与仓库面板节点
  2. 回退 `Main.cs` 仓库页绑定逻辑
  3. 保留报表卡片入口继续使用

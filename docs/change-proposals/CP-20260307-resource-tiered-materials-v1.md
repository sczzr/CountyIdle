# 改动提案（Change Proposal）

## 提案信息

- 标题：资源系统分层扩展（V1：T0-T1 原材料与加工链）
- 日期：2026-03-07
- 提案人：System Agent
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：
  - 现有资源链已覆盖“矿产开采 -> 金属锭 -> 新材料 -> 工业部件/建造构件”，但地图原料与加工品的边界仍偏抽象。
  - 在机制讨论中已经暴露出“青铜被误放入原材料层”的概念错误，说明资源层级命名仍需冻结。
  - 当前玩家更容易看到“库存有多少”，但不容易理解“自然采集层、工坊加工层、成品消费层”之间的关系。
- 证据（数据/玩家反馈）：
  - 本轮设计讨论已明确指出“青铜不是天然原材料，而是铜锭 + 锡锭的合金结果”。
  - 当前仓储系统虽已存在，但自然原材料、加工品、成品尚未形成稳定命名和阶段解锁规范。

## 改动内容

- 改什么：
  - 新增资源四层定义：`原材料层 / 一次加工层 / 二次加工层 / 成品层`
  - 冻结 `T0/T1` 的原材料、建筑链、产品链与显示边界
  - 在系统规格中明确“禁止进入原材料层”的项目，特别是 `青铜 / 玻璃 / 纸张 / 皮革 / 火药`
  - 将 `T0/T1` 资源接回三类消费端：建造、民生、工具
  - 新增“物品最小单位为 `1`”约束：可见库存无小数，分数变化进入隐藏进度池
  - 为后续 `T2/T3` 预留合金、轻工、火工与特产线，不在 V1 提前展开
- 不改什么：
  - 不在本阶段引入 `锡矿 / 青铜 / 竹材 / 石灰石 / 石英砂 / 纸张 / 灰浆 / 玻璃 / 火药 / 玉料 / 香木`
  - 不重写现有抽象资源汇总字段
  - 不直接改动战斗、探险、英雄实体系统
  - 不在本阶段强制接入完整价格系统与商路系统
- 影响系统：
  - `docs/02_system_specs.md`
  - `docs/05_feature_inventory.md`
  - `docs/08_development_list.md`
  - 后续实现将影响：
    - `CountyIdle/scripts/models/GameState.cs`
    - `CountyIdle/scripts/systems/ResourceSystem.cs`
    - `CountyIdle/scripts/systems/IndustrySystem.cs`
    - `CountyIdle/scripts/systems/EconomySystem.cs`
    - `CountyIdle/scripts/ui/WarehousePanel.cs`

## 预期结果

- 预期提升指标：
  - 玩家可在前期明确区分自然原材料与加工产物，降低资源理解门槛
  - `T0/T1` 资源能稳定支撑建造、民生、工具三类消费端，提升飞轮可解释性
  - 仓储面板、主界面资源栏与事件日志中的物品数量语义统一，不再出现“半个物品”
  - 后续引入 `青铜 / 纸张 / 玻璃 / 火药` 时不再发生层级混乱
  - 资源面板扩展有明确阶段边界，不会在前期信息过载
- 可接受副作用：
  - 前期资源栏与仓储面板条目数增加
  - 文档与代码字段之间会短期并存“细分表现层 + 抽象汇总层”双轨结构

## 验证计划

- 验证方式：
  - 审查 `T0/T1` 资源是否全部来自自然采集层
  - 审查 `铜矿 -> 铜锭`、`铁矿 -> 熟铁` 是否独立成立且不提前出现 `青铜`
  - 审查 `精盐 / 药剂 / 麻布 / 皮革` 是否都挂接到幸福、健康、衣物等民生反馈
  - 审查 UI 是否仅展示 `T0/T1` 已解锁资源
  - 审查可见库存、仓储面板、主要日志是否还出现物品小数
  - 后续实现阶段执行 `dotnet build .\Finally.sln` 与小时结算烟测
- 观察周期：设计冻结后至 V1 实现完成前；实现后观察 `10~20` 分钟挂机窗口
- 成功判定阈值：
  - 原材料层误置项为 `0`
  - `T0/T1` 闭环可解释
  - 未出现破坏旧存档和主循环的高风险改动

## 回滚条件

- 触发条件：
  - 资源细分导致 `T0/T1` 前期节奏明显卡死
  - 仓储与 UI 无法在不大改底层结构的前提下承载新条目
  - 细分链路影响旧存档兼容或小时结算稳定性
- 回滚步骤：
  1. 保留系统规格与功能卡归档
  2. 退回“表现层细分、底层继续汇总”的兼容方案
  3. 将 `T1` 只保留 `铜锭 / 熟铁 / 精盐 / 药剂 / 麻布 / 皮革` 六项核心产物
  4. 延后 `T2/T3` 的全部细分内容

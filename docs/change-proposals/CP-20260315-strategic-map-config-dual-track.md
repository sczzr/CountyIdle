# 改动提案（Change Proposal）

## 提案信息

- 标题：战略地图配置驱动双轨切换
- 日期：2026-03-15
- 提案人：Codex
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：战略地图已具备配置文件与回退加载，但主链仍优先使用程序化生成，缺少可对比的双轨切换与调试入口。
- 证据（数据/玩家反馈）：功能清单与开发列表均标记“配置驱动接入仍需双轨切换与调试入口”。

## 改动内容

- 改什么：
  - 在战略地图视图加入“配置驱动/生成驱动”切换入口（调试热键）。
  - 允许世界图与外域图在同一套配置定义与生成定义之间切换，便于验证配置效果。
- 不改什么：
  - 不移除现有程序化生成器。
  - 不改世界图的玩法事件/战斗结算逻辑。
- 影响系统：`StrategicMapViewSystem`、`StrategicMapConfigSystem`、`WorldPanel`（仅调试入口）。

## 预期结果

- 预期提升指标：配置驱动的地图可视化验证效率提升，地图调整无需反复改代码。
- 可接受副作用：调试入口仅开发态使用，正式体验仍以稳定路径为主。

## 验证计划

- 验证方式：手动切换并观察世界图/外域图渲染是否从 `strategic_maps.json` 读取。
- 观察周期：当天验证。
- 成功判定阈值：切换后地图几何/节点/线路随配置变化；配置缺失时仍可回退。

## 回滚条件

- 触发条件：切换导致崩溃或地图不可见。
- 回滚步骤：关闭调试切换入口，恢复仅生成驱动路径。

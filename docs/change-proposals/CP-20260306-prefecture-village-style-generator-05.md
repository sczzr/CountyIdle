# 改动提案（Change Proposal）

> 历史兼容说明（2026-03-08）：本文保留旧阶段的“县城 / 郡图 / 郡县 / 州府”等表述以便追溯；在当前项目语义中，请分别按“天衍峰驻地 / 江陵府外域 / 天衍峰经营 / 世界地图”理解。
> 当前对外语义：江陵府外域 / 外域附庸据点链路（保留历史 prefecture 技术名）


## 提案信息

- 标题：江陵府外域街区级高倍率与缩放惯性增强
- 日期：2026-03-06
- 提案人：Gameplay Agent
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：虽然已经能放大看到建筑足迹，但用户还需要更细的街屋块面，并要求缩放带惯性。
- 证据（数据/玩家反馈）：用户明确要求“最大缩放时的坊市再细化成一排排街屋块面，并且在进行缩放的时候需要带惯性”。

## 改动内容

- 改什么：
  - 将江陵府外域缩放改为平滑目标缩放
  - 高倍率下把 `ward` 渲染成成排街屋块面
  - 高倍率下把 `landmark/city` 渲染成院落式建筑群
- 不改什么：
  - 不调整经济/人口公式
  - 不新增新的地图交互玩法
- 影响系统：
  - `StrategicMapViewSystem`
  - `PrefectureMapGeneratorSystem`
  - `docs/02_system_specs.md`

## 预期结果

- 预期提升指标：
  - 缩放交互更顺滑
  - 江陵府外域最大缩放下建筑辨识度进一步提升
- 可接受副作用：
  - 高倍率渲染成本略有上升

## 验证计划

- 验证方式：
  - 连续滚轮缩放观察是否平滑过渡
  - 放大到最大观察街屋与院落块面
  - 执行 `dotnet build .\Finally.sln`
- 观察周期：5~10 分钟
- 成功判定阈值：缩放有惯性且街区级建筑可读

## 回滚条件

- 触发条件：缩放滞后严重或高倍率下街区过度拥挤
- 回滚步骤：
  1. 回退 `StrategicMapViewSystem` 惯性缩放逻辑
  2. 回退高倍率街区级建筑渲染
  3. 恢复上一版足迹渲染

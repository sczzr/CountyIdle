# 改动提案（Change Proposal）

## 提案信息

- 标题：仓储卡槽正式贴图与紧缺筛选增强
- 日期：2026-03-08
- 提案人：Gameplay Agent
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：上一版仓储卡槽已经完成分区与数量展示，但图标仍以 emoji 徽章为主，材料辨识度与“缺什么”反馈还不够强。
- 证据（数据/玩家反馈）：当前对话继续要求“继续”，且上一轮已明确给出“正式材料贴图 + 短缺高亮 / 分类筛选”作为下一步方向。

## 改动内容

- 改什么：
  - 为仓储卡槽补正式材料图片资源
  - 为每种材料定义短缺阈值，并在 UI 中显示“紧缺 / 缺货”状态
  - 增加顶部筛选按钮，支持按分类和“紧缺”查看
  - 保留现有仓储负载条、材料链扩建与快捷操作区
- 不改什么：
  - 不改仓储公式、资源产量、库存整数化规则
  - 不改主界面地图、岗位、存档等其他系统
  - 不引入排序、拖拽、收藏等新交互
- 影响系统：
  - `CountyIdle/assets/ui/materials/*`
  - `CountyIdle/scenes/ui/WarehousePanel.tscn`
  - `CountyIdle/scripts/ui/WarehousePanel.cs`
  - `docs/02_system_specs.md`

## 预期结果

- 预期提升指标：
  - 玩家通过图片可更快辨认材料类别
  - 紧缺物资定位效率明显高于上一版纯数量卡槽
  - 仓储分层成果在 UI 上更接近可玩的“背包 / 仓储”体验
- 可接受副作用：
  - 新增贴图资源后首次打开编辑器时会触发资源导入

## 验证计划

- 验证方式：
  - `dotnet build .\Finally.sln`
  - 打开仓储弹窗检查图片加载、筛选切换与状态高亮
  - 验证零库存 / 低库存 / 正常库存的视觉区分是否稳定
- 观察周期：一次构建 + 一轮交互烟测
- 成功判定阈值：无空引用、无资源路径错误、筛选状态刷新正常

## 回滚条件

- 触发条件：图片加载大面积失败、筛选逻辑导致卡槽显示异常，或仓储弹窗交互明显退化
- 回滚步骤：
  1. 回退 `WarehousePanel` 的贴图与筛选逻辑
  2. 移除新增材料贴图引用
  3. 恢复上一版卡槽网格

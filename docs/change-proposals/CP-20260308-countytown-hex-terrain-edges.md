# 改动提案（Change Proposal）

## 提案信息

- 标题：县城道路与水域边界补强为 hex 地形语义
- 日期：2026-03-08
- 提案人：Gameplay Agent
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：县城地表、建筑底座虽然已切到 hex 俯视，但道路与水域仍主要依赖底色区分，缺少更清晰的 hex 地形边界与通路语义。
- 证据（数据/玩家反馈）：上一轮县城 hex 俯瞰已经落地，下一步最自然的补强点就是道路连接感与水域岸线表现。

## 改动内容

- 改什么：
  - 为道路格增加 hex 核心道路面与相邻道路连接带
  - 为水域格增加内层水面、岸线高亮与轻量波纹
  - 保留现有 `TownMapGeneratorSystem` 与道路 / 水域布置逻辑，只增强视觉语义
- 不改什么：
  - 不改生成器的四邻域逻辑与岗位 / 居民结算
  - 不改建筑 / 场所 / 居民的核心链路
  - 不做真正逐格 tile 资源替换
- 影响系统：
  - `CountyIdle/scripts/systems/CountyTownMapViewSystem.cs`
  - `docs/02_system_specs.md`
  - `docs/05_feature_inventory.md`
  - `docs/08_development_list.md`

## 预期结果

- 预期提升指标：
  - 道路在县城页中更像连续的 hex 通路，而不是单格底色
  - 水域边界更清晰，玩家能更容易识别池塘 / 水面范围
  - 不破坏建筑、场所、居民与经营状态联动
- 可接受副作用：
  - 这轮仍基于现有四邻域逻辑派生连接带，不是完整 hex 邻接生成器

## 验证计划

- 验证方式：
  - `dotnet build .\Finally.sln`
  - 运行 Godot，检查县城页道路连续感、水域边界、缩放和经营态染色
- 观察周期：单次 5~10 分钟县城页验证
- 成功判定阈值：无构建错误、道路连接明显、无水域边界错位

## 回滚条件

- 触发条件：道路连接带遮挡严重、水域高亮过噪、县城性能明显退化
- 回滚步骤：
  1. 回退 `CountyTownMapViewSystem.cs` 中的地形语义 overlay
  2. 回退规格与开发记录中的道路 / 水域 hex 语义表述

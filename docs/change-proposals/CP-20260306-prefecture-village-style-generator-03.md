# 改动提案（Change Proposal）

## 提案信息

- 标题：开封城城门与坊市命名数据化
- 日期：2026-03-06
- 提案人：Gameplay Agent
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：郡图命名写死在代码中，调风格或本地化需要改代码重新发布。
- 证据（数据/玩家反馈）：用户明确要求“城门/坊市名称可配置 JSON”。

## 改动内容

- 改什么：
  - 新增 `prefecture_city_theme.json` 承载命名主题
  - 新增主题配置加载与容错系统
  - 郡图生成器读取配置驱动标签与建筑命名
- 不改什么：
  - 不改经济/人口/战斗数值
  - 不改地图交互逻辑
- 影响系统：
  - `scripts/models`
  - `scripts/systems`
  - `data/`
  - `docs/02_system_specs.md`

## 预期结果

- 预期提升指标：
  - 命名变更从“改代码”降为“改 JSON”
  - 城门/坊市/地标文案可快速换皮
  - 旧配置或缺失配置可安全回退
- 可接受副作用：
  - 配置项增多，维护需要遵循字段约束

## 验证计划

- 验证方式：
  - 修改 `prefecture_city_theme.json` 中门名后检查显示
  - 执行 `dotnet build .\Finally.sln`
  - 删除/破坏部分配置字段确认回退
- 观察周期：单轮 5~10 分钟
- 成功判定阈值：命名可配置且无新增编译错误

## 回滚条件

- 触发条件：配置解析异常导致地图命名丢失或运行告警过多
- 回滚步骤：
  1. 回退主题配置读取系统
  2. 回退生成器中的配置依赖
  3. 恢复硬编码默认命名

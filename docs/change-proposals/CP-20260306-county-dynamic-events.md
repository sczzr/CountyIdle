# 改动提案（Change Proposal）

> 历史兼容说明（2026-03-08）：本文保留旧阶段的“县城 / 郡图 / 郡县 / 州府”等表述以便追溯；在当前项目语义中，请分别按“天衍峰驻地 / 江陵府外域 / 天衍峰经营 / 世界地图”理解。
> 当前对外语义：宗门见闻 / 山门动态事件


## 提案信息

- 标题：宗门见闻动态事件增强
- 日期：2026-03-06
- 提案人：Gameplay Agent
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：主循环反馈主要来自固定公式，短周期“惊喜/风险”不足，职业分配与威胁变化的可感知度偏弱。
- 证据（数据/玩家反馈）：当前日志核心来源为探险、繁育与存档行为，缺少围绕坊市 / 讲法 / 护持秩序的宗门见闻事件层。

## 改动内容

- 改什么：
  - 新增 `CountyEventSystem`，每小时基于条件判定候选事件并按权重触发
  - 新增 `EventCooldownHours`，控制事件频率，防止连续刷屏
  - 新增 3 类事件：`商路集市`、`学宫讲习`、`边境袭扰`
  - 事件效果直接作用于 `Food/Gold/Research/Happiness/Threat`
- 不改什么：
  - 不新增事件分支选择 UI
  - 不改现有人口、科研、探险主公式
  - 不改存档结构版本号
- 影响系统：
  - `models/GameState`
  - `systems/CountyEventSystem`（新增）
  - `core/GameLoop`
  - `docs/02_system_specs.md`

## 预期结果

- 预期提升指标：
  - 10~15 分钟内至少出现 1 次事件日志
  - 商人与学者岗位占比提高时，正向事件出现频率可感知提升
  - 高威胁情况下出现负向反馈，驱动玩家继续探险控威胁
- 可接受副作用：
  - 资源短时波动增大，可能导致新手对因果感知成本上升

## 验证计划

- 验证方式：
  - `dotnet build .\Finally.sln`
  - 运行后观察 10 分钟以上日志，确认正负事件均可触发
  - 存档后读档确认新增字段兼容
- 观察周期：单次 10~20 分钟挂机窗口
- 成功判定阈值：无负资源异常、无崩溃、日志可见事件反馈

## 回滚条件

- 触发条件：事件导致核心飞轮被噪声打断（资源波动不可控或频率失衡）
- 回滚步骤：
  1. 停用 `GameLoop` 中 `CountyEventSystem` 调度
  2. 删除事件系统文件并回退 `GameState` 新字段
  3. 记录回滚原因并在下一轮提案调整阈值/权重

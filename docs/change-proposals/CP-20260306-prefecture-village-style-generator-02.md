# 改动提案（Change Proposal）

## 提案信息

- 标题：周边郡图开封城风格大尺度与高密建筑迭代
- 日期：2026-03-06
- 提案人：Gameplay Agent
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：虽然已有生态与城建表达，但用户反馈“地图仍不够大、建筑仍偏少”，希望参照古代开封城的繁盛度与空间结构。
- 证据（数据/玩家反馈）：本轮明确需求为“大地图 + 更多建筑 + 开封城参考”。

## 改动内容

- 改什么：
  - 扩大郡图边界生成参数与默认显示尺度
  - 城内建筑改为“地标 + 坊市网格”高密生成
  - 增加开封风格标签（主街、城门、坊市等）
  - 标签支持按缩放阈值显示（远景去噪，近景细节）
- 不改什么：
  - 不改主循环结算顺序与经济数值
  - 不增加地图交互逻辑
- 影响系统：
  - `PrefectureMapGeneratorSystem`
  - `StrategicMapViewSystem`
  - `StrategicMapConfig` / `StrategicMapConfigSystem`
  - `docs/02_system_specs.md`

## 预期结果

- 预期提升指标：
  - 郡图视觉范围更大，边界完整度提升
  - 城内建筑节点数量达到“高密坊市”观感
  - 缩放时标签密度可控，不因信息量增大而崩读
- 可接受副作用：
  - 放大视图下渲染对象数量上升，存在轻微性能压力

## 验证计划

- 验证方式：
  - 在郡图页验证默认视角范围、建筑密度与道路结构
  - 逐级缩放验证标签分级显隐
  - 执行 `dotnet build .\Finally.sln`
- 观察周期：10~15 分钟地图页连续观察
- 成功判定阈值：构建通过，且地图视觉满足“大且密”的目标

## 回滚条件

- 触发条件：出现严重遮挡、性能明显下降或可读性低于上一版
- 回滚步骤：
  1. 回退生成器大尺度与高密节点参数
  2. 回退标签分级显示字段与渲染过滤
  3. 恢复 `prefecture-village-style-generator-01` 版本

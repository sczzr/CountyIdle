# 改动提案（Change Proposal）

## 提案信息

- 标题：多存档面板筛选排序（V1）
- 日期：2026-03-08
- 提案人：Gameplay Agent
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：
  - 多槽面板已经能展示更多摘要，但槽位数量增多后，玩家仍需要手动扫完整个列表；
  - 自动档、主存档、手动档混在一起时，覆盖/读取目标定位效率偏低。
- 证据（数据/玩家反馈）：
  - `docs/05_feature_inventory.md` 已把“筛选排序”列为多存档界面的下一步差距。

## 改动内容

- 改什么：
  - 在 `SaveSlotsPanel` 增加筛选与排序控件
  - 将槽位显示切为全量数据与可见列表双层结构
  - 列表标题显示当前可见数/总数，并在无结果时给出提示
- 不改什么：
  - 不改变 SQLite 表结构
  - 不改变快速存档 / 快速读档与自动存档轮换规则
  - 不实现截图预览与云同步
- 影响系统：
  - `CountyIdle/scenes/ui/SaveSlotsPanel.tscn`
  - `CountyIdle/scripts/ui/SaveSlotsPanel.cs`
  - `docs/02_system_specs.md`

## 预期结果

- 预期提升指标：
  - 玩家更快定位目标存档
  - 自动档与手动档切换查看成本下降
  - 为后续截图预览与更复杂管理入口留出空间
- 可接受副作用：
  - 面板顶部控件增加，横向空间略紧

## 验证计划

- 验证方式：
  - 打开多存档面板并切换各筛选选项
  - 切换不同排序方式并确认列表顺序变化
  - 在筛选后执行读取/覆盖/新建/重命名/删除流程
  - `dotnet build .\Finally.sln`
- 观察周期：一轮启动 + 多次打开面板交互
- 成功判定阈值：
  - 列表、详情、按钮状态始终一致
  - 保护槽规则不受筛选排序影响

## 回滚条件

- 触发条件：
  - 面板切换筛选后出现选中错位或按钮状态异常
  - UI 压缩到影响现有操作可读性
- 回滚步骤：
  1. 回退筛选排序控件与可见列表逻辑
  2. 保留当前摘要预览与自动存档能力

# 改动提案（Change Proposal）

## 提案信息

- 标题：周边郡图生态地貌与城内建筑可视增强
- 日期：2026-03-06
- 提案人：Gameplay Agent
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：郡图建筑点位偏少，且缺少森林/湖泊/农田/山脉等环境描述，导致“周边郡图”可读性不足。
- 证据（数据/玩家反馈）：用户明确反馈“建筑太少、周边环境没有描述、城内建筑需要体现”。

## 改动内容

- 改什么：
  - 郡图程序化生成增加生态区域分层（森林、农田、山脉、湖泊）
  - 增加标签绘制能力，用于地貌与道路/城建说明
  - 增加郡城内建筑节点，并提高聚落节点与道路密度
- 不改什么：
  - 不变更小时结算顺序与核心数值系统
  - 不新增地图交互编辑系统
- 影响系统：
  - `StrategicMapConfig` 数据模型
  - `StrategicMapViewSystem` 渲染层
  - `PrefectureMapGeneratorSystem` 生成逻辑

## 预期结果

- 预期提升指标：
  - 郡图可辨识 5 类以上地貌/人工区域信息
  - 郡城内可见多建筑节点
  - 聚落与道路网络相较上一版更密集
- 可接受副作用：
  - 地图文字说明增加导致边缘区域略显拥挤

## 验证计划

- 验证方式：
  - 运行地图页观察自然/人工区域是否可辨识
  - 检查郡城中心建筑节点显示与缩放后可读性
  - 执行 `dotnet build .\Finally.sln` 记录编译结果
- 观察周期：单轮 10 分钟地图观察
- 成功判定阈值：地图表达提升且无新增编译错误

## 回滚条件

- 触发条件：标签/节点拥挤导致主要信息不可读，或渲染异常明显增多
- 回滚步骤：
  1. 回退生成器新增生态与建筑节点逻辑
  2. 回退渲染器标签绘制能力
  3. 保留上一版程序化郡图主干

# 改动提案：双地图布局（宗门地图 + 世界地图）

## 提案信息

- 标题：主地图入口收敛为宗门地图与世界地图
- 日期：2026-03-08
- 提案人：Codex
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：主地图区域混合了县城、郡图、世界图、事件、报表、探险等多页签，入口过多，不符合“只保留宗门地图与世界地图”这一新的产品方向。
- 证据（数据/玩家反馈）：本轮需求明确要求“总共就两个地图，一个是宗门地图，一个是世界地图”。

## 改动内容

- 改什么：
  - 主地图区域只保留 `宗门地图 / 世界地图` 两个可见入口；
  - 现有县城 hex 驻地视图统一转为宗门地图语义；
  - 世界图统一以修仙世界世界图语义显示。
- 不改什么：
  - 不新增新的宗门地图生成器；
  - 不重写仓储弹窗与侧栏 UI；
  - 不改 `GameState` 与小时结算。
- 影响系统：
  - `CountyIdle/scripts/Main.cs`
  - `CountyIdle/scenes/ui/WorldPanel.tscn`
  - `CountyIdle/scripts/systems/CountyTownMapViewSystem.cs`
  - `CountyIdle/scripts/systems/StrategicMapViewSystem.cs`

## 预期结果

- 预期提升指标：
  - 主地图入口认知负担降低；
  - 宗门经营与世界形势成为固定双核心视角；
  - 地图缩放与经营状态条继续沿用，不增加交互复杂度。
- 可接受副作用：
  - 原郡图/事件/报表/探险页签不再占据主地图入口；
  - 宗门地图当前仍复用原县城 hex 驻地视图实现。

## 验证计划

- 验证方式：
  - `dotnet build .\Finally.sln`
  - 检查 `WorldPanel.tscn` 顶部可见页签只剩双地图
- 观察周期：本次迭代即时验证
- 成功判定阈值：
  - 双地图模式能正常编译；
  - 默认地图页 = 宗门地图；
  - 世界地图和宗门地图缩放继续可用。

## 回滚条件

- 触发条件：地图页签无法切换、宗门图/世界图不可见、构建失败。
- 回滚步骤：
  1. 恢复 `WorldPanel.tscn` 中被隐藏的旧页签按钮；
  2. 恢复 `Main.cs` 中旧的多页签切换逻辑；
  3. 保留宗门地图文案调整，后续再评估双地图方案。

# 改动提案（Change Proposal）

## 提案信息

- 标题：主场景按钮全量可交互修复
- 日期：2026-03-06
- 提案人：Codex
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：主场景存在可见按钮缺少交互绑定，或处于不可点击状态，形成 UI/UX 断点。
- 证据（数据/玩家反馈）：用户反馈“场景中的所有按钮交互都需要能够起作用，不能出现不能交互的 UI/UX”。

## 改动内容

- 改什么：
  - 为岗位优先按钮补齐事件，点击后可切换优先态并写入日志
  - 为矿坑按钮补齐事件，未解锁给出提示，解锁后执行扩建行为
  - 取消矿坑按钮默认禁用，避免出现不可点击按钮
- 不改什么：
  - 不调整核心结算频率与核心飞轮公式
  - 不新增矿业独立子系统
- 影响系统：`CountyIdle/scripts/Main.cs`、`CountyIdle/scenes/ui/WorldPanel.tscn`

## 预期结果

- 预期提升指标：
  - 主场景可见按钮“点击无反馈”问题降至 0
  - 交互反馈覆盖到岗位优先与矿坑卡片入口
- 可接受副作用：
  - 岗位优先当前为 UI 调度反馈，暂不直接改动产出公式

## 验证计划

- 验证方式：
  - `dotnet build .\Finally.sln`
  - Godot 运行主场景手点验证按钮反馈
- 观察周期：单次会话（5~10 分钟）
- 成功判定阈值：主场景可见按钮点击均有状态变化或日志反馈，且无运行时异常

## 回滚条件

- 触发条件：按钮交互导致状态异常、存读档异常或主循环异常。
- 回滚步骤：回退 `Main.cs` 与 `WorldPanel.tscn` 本次改动并恢复前版绑定逻辑。

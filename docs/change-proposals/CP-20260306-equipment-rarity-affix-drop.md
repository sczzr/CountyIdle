# 改动提案（Change Proposal）

## 提案信息

- 标题：探险装备品质与词条掉落（M2 成长深度补齐）
- 日期：2026-03-06
- 提案人：Codex
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：探险胜利主要回流金币与稀有素材，装备成长反馈弱，`AvgGearScore` 增长来源单一。
- 证据（数据/玩家反馈）：现有 `CombatSystem` 仅在胜负与层数上形成反馈，缺少“战利品质量”维度，武装探险环的可期待感不足。

## 改动内容

- 改什么：
  - 新增装备掉落规则：胜利后按概率掉落装备，包含品质（普通/精良/史诗/传说）与词条（概率触发）
  - 掉落结果转换为 `AvgGearScore` 增量并写日志
  - `GameState` 新增四档装备计数字段用于展示与存档
- 不改什么：
  - 不增加主动锻造界面与装备槽管理
  - 不改人口、经济、研究主流程公式
- 影响系统：
  - `models/GameState`
  - `systems/CombatSystem`、`systems/EquipmentSystem`
  - `scripts/Main.cs`（展示文案）

## 预期结果

- 预期提升指标：
  - 每次胜利“可感知奖励事件”比例提升（新增装备掉落日志）
  - 平均战备评分在同窗口内更平滑增长
  - 探险推进期望略提升但不应破坏失败压力
- 可接受副作用：
  - 中期胜率可能略升，需要后续微调品质权重

## 验证计划

- 验证方式：
  - 编译校验 + 运行后观察探险日志、装备计数与评分增长
- 观察周期：
  - 单局连续 20~30 分钟（对应 20~30 游戏小时）
- 成功判定阈值：
  - 日志出现装备品质/词条记录
  - `AvgGearScore` 单调非负增长，不出现异常爆炸
  - 不出现存档读档错误

## 回滚条件

- 触发条件：
  - 掉落增益导致探险失败率显著下降且威胁长期贴近 0
  - 存档兼容性出现异常
- 回滚步骤：
  1. 取消 `CombatSystem` 对装备掉落调用
  2. 移除 `EquipmentSystem` 与 `GameState` 新增字段引用
  3. 恢复旧版展示文案并保留本提案记录

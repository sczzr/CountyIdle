# 改动提案（Change Proposal）

## 提案信息

- 标题：县城地图程序化生成与 2.5D 中式视角落地
- 日期：2026-03-06
- 提案人：Gameplay Agent
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：县城地图仅为占位文本，无法承载“县域经营”的空间感，也难以体现中式题材特色。
- 证据（数据/玩家反馈）：现有三层地图中“县城地图”缺乏可视化信息，地图切页可交互但内容不可感知。

## 改动内容

- 改什么：
  - 新增 `TownMapGeneratorSystem`：按种子生成道路骨架、地块类型与房屋落位
  - 新增县城地图数据模型（地形类型、房屋朝向/层数等）
  - 新增 `CountyTownMapViewSystem`：在县城页绘制 2.5D 斜面地图与中式房屋风格
  - 增加“重生成”入口，便于快速刷新地图样式
- 不改什么：
  - 不调整核心飞轮结算逻辑（人口/职业/科研/探险）
  - 不改存档结构与读档流程
  - 不引入地图交互玩法（寻路、点击编辑、战斗网格）
- 影响系统：
  - `scripts/models`（新增地图模型）
  - `scripts/systems`（新增地图生成与渲染脚本）
  - `scripts/Main.cs`（状态同步调用）
  - `scenes/ui/WorldPanel.tscn`（县城地图页结构）
  - `docs/02_system_specs.md`

## 预期结果

- 预期提升指标：
  - 县城页首次打开即有可视化地图反馈（非文本）
  - 手动重生成后布局可见变化（道路/房屋分布）
  - 人口变化后房屋密度同步变化
- 可接受副作用：
  - 地图仅做展示，不与建筑系统数据逐栋绑定

## 验证计划

- 验证方式：
  - `dotnet build .\Finally.sln`
  - 运行 Godot，进入“县城地图”确认 2.5D 渲染与中式房屋风格
  - 点击“重生成”确认布局刷新
- 观察周期：单次 5~10 分钟 UI 交互验证
- 成功判定阈值：无报错、无负值副作用、主循环与存档入口不回归

## 回滚条件

- 触发条件：地图渲染导致明显卡顿或 UI 逻辑回归（按钮失效/切页异常）
- 回滚步骤：
  1. 回退 `WorldPanel.tscn` 的县城地图节点结构
  2. 删除新增地图模型与系统脚本
  3. 回退 `Main.cs` 对县城地图视图的绑定逻辑

# 改动提案（Change Proposal）

## 提案信息

- 标题：郡图御街-坊巷高倍细化增强
- 日期：2026-03-06
- 提案人：Gameplay Agent
- 变更级别：`L2 机制增强`

## 改动背景

- 当前问题：郡图已支持大图、惯性缩放与高倍建筑块面，但高倍状态下仍偏“单点式建筑”，整体城市肌理不够像开封城的坊市排布。
- 用户诉求：继续细化“最大缩放时”的郡城，让建筑数量更多，能够看到主街、坊巷和一排排街屋。

## 改动内容

- 改什么：
  - 提高郡城 `ward` 网格密度
  - 增加城内竖向御街、横街与次级里巷数量
  - 在高倍渲染层叠加整座郡城的街巷纹理
  - 将 `ward` 进一步绘制为沿街屋面、坊内庭院与市肆摊列
- 不改什么：
  - 不改变 `GameState` 数值结算
  - 不新增建筑交互与经营联动逻辑
- 影响系统：
  - `CountyIdle/scripts/systems/PrefectureMapGeneratorSystem.cs`
  - `CountyIdle/scripts/systems/StrategicMapViewSystem.cs`
  - `docs/02_system_specs.md`

## 预期结果

- 郡城最大缩放下更接近“开封城式主街-坊巷-街屋”的视觉体验
- 同一倍率下能观察到更多成片建筑，而不是少量离散节点
- 保持已有惯性缩放与标签分级显隐能力

## 验证计划

- 连续滚轮缩放，观察主街与里巷是否随高倍状态清晰出现
- 在最大缩放下观察坊市节点是否形成成排街屋/市肆/院落
- 执行 `dotnet build .\Finally.sln`

## 回滚条件

- 触发条件：高倍状态下城市可读性明显下降，或新增绘制导致明显卡顿
- 回滚步骤：
  1. 回退高倍街巷纹理叠加
  2. 回退高密坊市节点与沿街屋面绘制
  3. 保留既有惯性缩放与基础高倍建筑表现

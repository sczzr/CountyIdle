# 改动提案（Change Proposal）

## 提案信息

- 标题：主界面交互重连与按钮功能化
- 日期：2026-03-06
- 提案人：Codex
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：主界面可见按钮存在“展示态”但未接入玩法事件，玩家点击后无有效反馈。
- 证据（数据/玩家反馈）：用户反馈“当前主界面不能进行交互，点击按钮没有反应”。

## 改动内容

- 改什么：
  - 对主界面两套布局（旧布局/Figma 布局）建立可运行的按钮事件绑定
  - 将倍速按钮接入真实时间倍率控制
  - 将 Figma 中央动作按钮接入产业扩建与制工具行为
- 不改什么：
  - 不改存档结构与核心资源公式
  - 不引入新系统依赖
- 影响系统：`scripts/Main.cs`、`scripts/core/GameLoop.cs`、`UI(figma)`

## 预期结果

- 预期提升指标：
  - 主界面关键按钮有效响应率从“部分无效”提升到“全部可触发”
  - 玩家 10 分钟内可完成至少一次“建造/制工具/存读档”交互闭环
- 可接受副作用：
  - Figma 与旧布局在文案呈现上允许存在轻微差异

## 验证计划

- 验证方式：
  - `dotnet build .\Finally.sln`
  - 启动主场景后手点验证：探险开关、倍速切换、存档/读档、产业动作按钮
- 观察周期：一次会话（5~10 分钟）
- 成功判定阈值：按钮点击均有状态或日志反馈，且无运行时异常

## 回滚条件

- 触发条件：任一主按钮触发后造成循环中断、存档异常或无法继续结算
- 回滚步骤：回退 `Main.cs` 与 `GameLoop.cs` 本次改动，恢复旧版交互路径

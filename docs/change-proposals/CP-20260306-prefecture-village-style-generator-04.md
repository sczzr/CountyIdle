# 改动提案（Change Proposal）

> 历史兼容说明（2026-03-08）：本文保留旧阶段的“县城 / 郡图 / 郡县 / 州府”等表述以便追溯；在当前项目语义中，请分别按“天衍峰驻地 / 江陵府外域 / 天衍峰经营 / 世界地图”理解。
> 当前对外语义：江陵府外域 / 外域附庸据点链路（保留历史 prefecture 技术名）


## 提案信息

- 标题：江陵府外域最大缩放与建筑足迹可视增强
- 日期：2026-03-06
- 提案人：Gameplay Agent
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：江陵府外域虽然有高密建筑，但最大缩放不足，放大后仍难以直观看清建筑形体。
- 证据（数据/玩家反馈）：用户明确提出“地图有点小了，放大后最大应该能够看到建筑的”。

## 改动内容

- 改什么：
  - 为 `Prefecture` 模式提高最大缩放上限
  - 为江陵府外域节点增加建筑类型（附庸据点/坊市/地标）
  - 在高缩放下按类型绘制建筑足迹
- 不改什么：
  - 不改世界图基础缩放体验
  - 不改地图数值系统
- 影响系统：
  - `StrategicMapConfig`
  - `StrategicMapViewSystem`
  - `PrefectureMapGeneratorSystem`

## 预期结果

- 预期提升指标：
  - 江陵府外域最大缩放达到可辨识建筑级别
  - 高缩放下建筑不再只是圆点
- 可接受副作用：
  - 高倍率下视图更依赖滚轮回退查看全貌

## 验证计划

- 验证方式：
  - 构建后在江陵府外域页滚轮放大到最大观察建筑足迹
  - 执行 `dotnet build .\Finally.sln`
- 观察周期：5~10 分钟
- 成功判定阈值：建筑足迹可见且构建通过

## 回滚条件

- 触发条件：高倍率表现明显破坏阅读体验
- 回滚步骤：
  1. 回退 `StrategicMapViewSystem` 缩放上限
  2. 回退建筑足迹渲染逻辑
  3. 恢复上一版节点显示

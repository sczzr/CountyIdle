# 改动提案（Change Proposal）

## 提案信息

- 标题：仓储弹窗卡槽化库存重构
- 日期：2026-03-08
- 提案人：Gameplay Agent
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：现有仓储弹窗主要依赖多行文本串联材料信息，玩家需要逐段阅读才能定位具体材料库存，视觉辨识度偏弱。
- 证据（数据/玩家反馈）：新增需求明确要求“重构仓库的 UI 界面，有各种格子显示具体材料的图片和数量”。

## 改动内容

- 改什么：
  - 将仓储库存区改为按类别分区的卡槽网格
  - 每个材料卡槽显示图标、名称、整数数量与提示信息
  - 保留并下移现有仓储快捷操作区，继续支持矿仓联建、制工具、扩建工坊、扩建庶务殿与四条 `T0` 链路
  - 零库存材料继续展示，但以弱化样式提示“已知但当前缺货”
- 不改什么：
  - 不改 `GameState`、存档结构与库存整数化规则
  - 不改小时结算顺序、资源公式与仓储容量公式
  - 不新增筛选/排序/拖拽整理等复杂交互
- 影响系统：
  - `CountyIdle/scenes/ui/WarehousePanel.tscn`
  - `CountyIdle/scripts/ui/WarehousePanel.cs`
  - `docs/02_system_specs.md`

## 预期结果

- 预期提升指标：
  - 玩家能更快识别关键材料瓶颈与库存结构
  - 仓储面板从“文本读数”提升为“扫视识别”，降低认知成本
  - 现有材料分层成果在 UI 上更直观可见
- 可接受副作用：
  - 弹窗会比旧版更高、更宽，并在库存较多时依赖滚动区域

## 验证计划

- 验证方式：
  - `dotnet build .\Finally.sln`
  - 打开仓储弹窗检查分类、卡槽数量、图标与整数库存刷新
  - 验证快捷操作按钮与 `Esc` 关闭行为未回归
- 观察周期：一次构建 + 一轮弹窗交互烟测
- 成功判定阈值：无空引用、无节点路径错误、无快捷操作失效

## 回滚条件

- 触发条件：仓储弹窗出现高频布局溢出、刷新报错或操作回归，导致主界面仓储入口不可用
- 回滚步骤：
  1. 回退 `WarehousePanel.tscn` 与 `WarehousePanel.cs`
  2. 恢复旧版文本分类展示
  3. 保留原有弹窗入口与事件绑定

# 改动提案：宗主治理中枢（一期：方略层去人头化）

- 任务 ID：`DL-037`
- 标题：把玩家交互从“人头调度”改成“宗主定方向”
- 改动级别：`L2 机制增强`

## 背景

当前 `天衍峰法旨` 虽然已经替代了直接调岗位，但玩家仍会在摘要与详情中看到“请求多少人 / 实派多少人 / 空闲多少人”这类执事层信息。  
这会让玩家的决策重心落到微观排班，而不是宗主应关心的宗门发展方向、法令力度、任务重点与人才培养。

## 本轮提案

1. 保留现有底层折算逻辑与双轨内务规则，避免破坏小时结算和存档兼容；
2. 将玩家可见交互改成“治理条目 + 侧重强度 + 执事落实态势”；
3. 将 `TaskPanel` 对外改为“宗主中枢”，按钮改成 `收敛 / 推进 / 鼎力推进 / 恢复均衡`；
4. 将 `JobsPadding` 改为“宗主治理摘要”，明确“宗主只定方向，执事自动排班”。

## 风险

- 玩家可能短期内不清楚“侧重强度”和原有法旨刻度的映射关系；
- 底层仍保留人数折算，如果日志与文档没有同步，容易出现语义割裂；
- 若后续不继续补门规 / 育才模块，宗主中枢仍会显得内容偏薄。

## 缓解

- 在中枢详情中明确写出“宗主定调，执事层自动落实”；
- 日志、按钮、摘要三处统一改词，避免继续暴露请求/实派人数；
- 将“门规 / 育才 / 季度法令”登记到后续治理中枢二期。

## 验收

- 玩家不再直接看到“请求人数 / 实派人数 / 空闲门人”；
- 中枢与摘要明确表达“治理方向 / 执行态势”；
- `dotnet build .\Finally.sln` 通过。

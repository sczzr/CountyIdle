# 改动提案：DL-013 地图与经营状态联动（一期）

> 历史兼容说明（2026-03-08）：本文保留旧阶段的“县城 / 郡图 / 郡县 / 州府”等表述以便追溯；在当前项目语义中，请分别按“天衍峰驻地 / 江陵府外域 / 天衍峰经营 / 世界地图”理解。
> 当前对外语义：天衍峰山门图 / 江陵府外域 / 世界地图与经营联动


## 提案信息

- 标题：地图与经营状态联动（一期）
- 日期：2026-03-06
- 提案人：Codex
- 变更级别：`L2 机制`

## 改动背景

- 当前问题：
  - 地图系统主要承担视觉展示，经营状态变化虽然会驱动部分布局重建，但玩家无法直观看出“当前宗门与江陵府外域是繁荣还是吃紧”。
  - 地图页缺少反向调度入口，玩家不能直接从地图侧介入灵道、坊市与附庸据点护持。
- 证据（数据/玩家反馈）：
  - `docs/08_development_list.md` 中 `DL-013` 长期处于 `TODO`；
  - 当前 `docs/05_feature_inventory.md` 将“地图与经营状态联动”列为未实现。

## 改动内容

- 改什么：
  - 新增地图联动系统，按经营状态为世界图、郡图、县城图生成状态样式；
  - 地图页新增状态条与双按钮调度入口；
  - 新增郡图/县城的即时调度效果（整修驿路、赈济乡里、修整街坊、夜巡清巷）。
- 不改什么：
  - 不引入新的地图页；
  - 不改变现有 `60 分钟` 小时结算顺序；
  - 不把 Figma 布局一起扩成完整地图调度界面。
- 影响系统：
  - `StrategicMapViewSystem`
  - `CountyTownMapViewSystem`
  - `GameLoop`
  - `WorldPanel`

## 预期结果

- 预期提升指标：
  - 玩家在地图页可立即识别当前经营压力；
  - 地图页具备至少两类可执行的治理动作；
  - 地图表现与经营指标形成双向闭环。
- 可接受副作用：
  - 地图颜色会出现轻微动态偏移；
  - 地图调度为即时数值反馈，一期不做更复杂的持续性 Buff。

## 验证计划

- 验证方式：
  - 手动切换世界/郡图/县城页观察状态文本与色调变化；
  - 执行地图调度并观察 `Happiness`、`Threat`、`AverageCommuteDistanceKm`、资源数值与日志变化；
  - 运行 `dotnet build .\Finally.sln`。
- 观察周期：单次会话即时验证 + 至少一个小时结算周期。
- 成功判定阈值：
  - 调度成功时即时刷新地图状态；
  - 调度失败时给出资源不足等日志；
  - 构建无错误。
